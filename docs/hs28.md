# Həll saatı 27. GCD, LCM və sadə ədədlər

Ədədlər nəzəriyyəsi olimpiadaların daimi qonağıdır. Bu dərsdə dörd təməl nəzəriyyə: Evklid alqoritmi (GCD), sadəlik yoxlaması, Eratosfen ələyi və sürətli qüvvətə yüksəltmə + mod arifmetikası haqqında danışacayıq.

## GCD: Evklid alqoritmi

Ən böyük ortaq bölən. Açar fakt: `gcd(a, b) = gcd(b, a % b)`, base case olaraq isə `gcd(a, 0) = a` götürülür:

```cpp
long long gcd(long long a, long long b) {
    while (b != 0) {
        a %= b;
        swap(a, b);
    }
    return a;
}
```

Niyə işləyir? a və b-nin hər ortaq böləni `a % b = a − k * b`-ni də bölür. Beləliklə, hər addımda ən azı 2 böləni gedir deyə ədəd ən azı 2 dəfə azalır, beləliklə O(log(min(a,b))) zaman səmərəliyinə işləyir, hansı 10¹⁸ üçün təxmini 90 addım edir.

:::note[Qeyd]
C++17-dən hazır `__gcd(a, b)` və `gcd(a, b)` (`<numeric>`)-da var.
:::

**LCM** (ən kiçik ortaq bölünən) GCD-dən çıxır:

```cpp
long long lcm(long long a, long long b) {
    return a / gcd(a, b) * b;
}
```

:::tip[Overflow-un qarşısını almaq]
Əvvəlcə GCD-sinə bölüb sonra vururuq deyə overflow-un qarşısını almış oluruq. `a * b / gcd` yazsanız, `a * b` daşa bilər, buna görə bölməni əvvələ çəkmək daha məntiqlidir.
:::


## Sadəlik yoxlaması: O(sqrt(n))

n-in bölənləri cüt-cüt gəlir: d bölürsə, n/d də bölür və cütün biri həmişə ≤ √n olur. Deməli √n-ə qədər yoxlamaq bəsdir:

```cpp
bool isprime(long long n) {
    if (n < 2) return false;
    for (long long d = 2; d * d <= n; d++)
        if (n % d == 0) return false;
    return true;
}
```

`d <= sqrt(n)`-i sqrt işlətmədən `d * d <= n` kimi yazmaq olar. `sqrt()` ağır əməliyyat olduğundan, ikinci yazılış kodu daha sürətli edir.
Eyni ideya ilə bölənləri saymaq / sadə vuruqlara ayırmaq:

```cpp
// n-in sadə vuruqlara ayrılışı
for (long long d = 2; d * d <= n; d++)
    while (n % d == 0) {
        cout << d << ' ';
        n /= d;
    }
if (n > 1) cout << n;
```

Sonuncu sətri unutmayın: 2 * 998244353 = (2 * 998244353)-ün 998244353-lük hissəsi kimi böyük sadə vuruğu dövr daxilində tapa bilmirik.

## Eratosfen ələyi: 1..n-in hamısı üçün

Çoxlu ədədin sadə olmağını yoxlamaq sorğusu varsa (məsələn, "10⁶-ya qədər hansılar sadədir?"), hər ədədi ayrıca yoxlamaq effektiv deyil. Bu zaman ələkdən istifadə edirik, hansı ki mürəkkəb ədədləri "ələyir", yəni işarələyir:

```cpp
const int N = 1000001;
vector<bool> pr(N, true);

void sieve() {
    pr[0] = pr[1] = 0;
    for (int i = 2; (long long)i * i < N; i++)
        if (pr[i])
            for (int j = i * i; j < N; j += i)
                pr[j] = 0;
}
```

Mürəkkəblik O(n log log n) olur, hansı ki praktikada n = 10⁷-yə qədər effektiv şəkildə hesablamağa bəs edir. `j = i * i`-dən başlamaq əslində bir optimizasiyadır, çünki daha kiçik mislər (2i, 3i, …) artıq kiçik sadələr tərəfindən silinib.

Ələyin istifadə hallarından biri də ən kiçik sadə vuruq massividir. Bu halda hər bir indeksdəki dəyər ədədin sadə vəya mürəkkəb olduğunu yox, həmin ədədin ən kiçik sadə bölənini göstərir. Bununla istənilən x ədədini O(log x) zaman səmərəliyində vuruqlara ayırmaq mümkündür:

```cpp
vector<int> prime(N, 0);
for (int i = 2; i < N; i++)
    if (prime[i] == 0)
        for (long long j = i; j < N; j += i)
            if (prime[j] == 0) prime[j] = i;

while (j > 1) { cout << prime[j] << ' '; j /= prime[j]; }
```

### Nümunə: T-primes (CF 230B)

"Düz 3 böləni olan ədədlər" sadələrin kvadratları (p² bölənləri: 1, p, p²) olanlardır. Həlli isə 10⁶-ya qədər sieve + hər sorğuda tam kvadrat və kökünün sadə olmağını yoxlamaqdır.

## Mod arifmetikası və sürətli qüvvət

```cpp
(a + b) % M = ((a % M) + (b % M)) % M
(a * b) % M = ((a % M) * (b % M)) % M
(a - b) % M = ((a % M) - (b % M) + M) % M
```

:::danger[Çıxmada +M]
C++-da `(-3) % 5 = -3` (müsbət deyil!). Fərqləri mod alanda həmişə `+ M` əlavə edin. Bu, mod məsələlərində 1 nömrəli buqdır.
:::

Sürətli qüvvətə yüksəltmə (binary exponentiation) aᵇ mod M-i O(log b)-də hesablamağa kömək edir:

```cpp
long long pw(long long a, long long b, long long M) {
    a %= M;
    long long cv = 1;
    while (b > 0) {
        if (b & 1) cv = cv * a % M;   // b-nin son biti 1-dirsə
        a = a * a % M;
        b >>= 1;
    }
    return cv;
}
```

Məntiq: b-ni ikilik yazılışına açırıq: a¹³ = a⁸ * a⁴ * a¹. Hər addımda a kvadratlanır, b-nin biti 1 olanda nəticəyə vurulur.

## Çalışmalar

**Çalışma 1.** n ədədin hamısının GCD-ni və LCM-ni tapın (LCM üçün ehtiyatlı olun, nəticə overflow alarsa nə etməliyik?).

<details>
<summary>Həlli</summary>

```cpp
long long g = 0;
for (long long x : a) g = gcd(g, x);
long long l = 1;
for (long long x : a) {
    long long g2 = gcd(l, x);
    if (l / g2 > 2e18 / x) { /* overflow ola bilər, məsələ mod istəyirsə mod-da hesablayın */ }
    l = l / g2 * x;
}
```

GCD monoton kiçilir, problem yoxdur. Lakin, LCM çox böyüyə bilər. Məsələ böyük LCM istəyirsə, adətən vuruqların maksimum dərəcələri ilə (və mod ilə) işləmək lazımdır.

</details>

**Çalışma 2.** `a/b` kəsrini ixtisar olunmuş şəkilə gətirin, sonra isə iki kəsri toplayın.

<details>
<summary>Həlli</summary>

```cpp
long long g = gcd(a, b);
cout << a / g << "/" << b / g;

// a/b + c/d:
long long p = a * d + c * b, q = b * d;
g = gcd(p, q);
cout << p / g << "/" << q / g;
```

Bu tipli kəsr arifmetikasından `double`-sız dəqiq hesablamaq üçün istifadə olunur.

</details>

**Çalışma 3.** Eratosfen ələyi mənqiti ilə 1-dən n-ə qədər (n ≤ 10⁶) hər ədədin bölənlərinin sayını çıxaran cədvəl qurun. Kod O(n log n) zamana işləsin.

<details>
<summary>Həlli</summary>

```cpp
vector<int> bolen(N, 0);
for (int d = 1; d < N; d++)
    for (int j = d; j < N; j += d)
        bolen[j]++;
```

Hər d öz qatlarına +1 verir, cəmi əməliyyat n/1 + n/2 + … + n/n ≈ n * ln olur, hansı ki, belə cəmə harmonik cəm deyirik.

</details>

## Tez-tez edilən səhvlər

1. LCM hesablayarkən overflow almaq;
2. Mənfi mod: çıxmadan sonra `+ M` unutmaq;
3. Sadəlik yoxlamasında `d <= sqrt(n)` kimi yazmaq;
4. Vuruqlara ayırmada qalan böyük sadəni (`n > 1`) unutmaq;
5. Ələk massivini hər testdə yenidən qurmaq.
