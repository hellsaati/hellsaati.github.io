Həll saatı 18. Heap və priority queue

Tez-tez belə vəziyyət yaranır: elementlər gəlir, gedir, və hər an bizə ən böyüyü (və ya ən kiçiyi) lazımdır. Hər dəfə sıralamaq — O(n log n) hər sorğuda, çox bahadır. priority_queue (daxildə heap) bunu belə edir:

  • ən böyüyə baxmaq — O(1)
  • əlavə etmək — O(log n)
  • ən böyüyü çıxarmaq — O(log n)

priority_queue əsasları​

#include <queue>

priority_queue<int> pq;
pq.push(3);
pq.push(10);
pq.push(7);

cout << pq.top();
pq.pop();
cout << pq.top();

15-ci dərsdəki queue-dan fərqi: queue gəlmə sırası ilə verir (FIFO), priority_queue — qiymətə görə.

Min-heap​

Ən kiçiyi üstə qoymaq üçün standart yazılış:

priority_queue<int, vector<int>, greater<int>> pq;   // MIN-heap

Yazılış qorxunc görünür — əzbərləyin, hamı belə edir. Alternativ hiylə: qiymətləri mənfi işarə ilə max-heap-ə atmaq (pq.push(-x), oxuyanda -pq.top()).

Heap daxildə necə işləyir?

Heap — hər atanın öz uşaqlarından böyük (max-heap) olduğu binar ağacdır, massivdə saxlanır: i elementinin uşaqları 2i+1 və 2i+2. Əlavə/çıxarma zamanı element yuxarı-aşağı "üzür" — hündürlük log n olduğundan əməliyyatlar O(log n). STL bunu sizin üçün edir; quruluşu bilmək müsahibə və nəzəriyyə üçün faydalıdır.

Klassik template-lər​

1. k ən böyük element​

n elementdən k ən böyüyü lazımdır (n böyük, k kiçik). k ölçülü min-heap saxlayırıq:

priority_queue<int, vector<int>, greater<int>> pq;
for (int i = 0; i < n; i++) {
    int x;
    cin >> x;
    pq.push(x);
    if ((int)pq.size() > k) pq.pop();   // ən kicik cixir
}
// pq-da indi k ən böyük element var; pq.top() — onlarin ən kicigi (k-ci ən böyük)

Mürəkkəblik O(n log k) — tam sıralamadan yaxşıdır. Məntiq: heap-də həmişə "hazırkı ən yaxşı k" yaşayır; yeni gələn ən zəifi sıxışdırır.

2. Simulyasiya: hər dəfə ən yaxşısını götür​

"Hər addımda ən böyük iki ədədi götür, birləşdir, geri qoy" tipli məsələlər:

// Numune: dashlarin toqquşması — ən böyük iki dash toqquşur,
// ferqleri qalir (0-dan ferqlidirse geri qayidir)
priority_queue<long long> pq;
for (int i = 0; i < n; i++) {
    long long x;
    cin >> x;
    pq.push(x);
}

while (pq.size() > 1) {
    long long a = pq.top(); pq.pop();
    long long b = pq.top(); pq.pop();
    if (a != b) pq.push(a - b);
}
cout << (pq.empty() ? 0 : pq.top());

3. Pair-lərlə: qiymət + kimlik​

Çox vaxt "ən yaxşı"nın kim olduğu da lazımdır:

priority_queue<pair<int,int>> pq;    // {bal, oyuncu_id}
pq.push({95, 3});
pq.push({87, 1});

auto [bal, id] = pq.top();           // 95, 3 — pair-ler first-e gore muqayise olunur

Bu template Dijkstra alqoritminin (gələcək mövzu) təməlidir: {məsafə, şəhər} cütləri min-heap-də.

Nümunə: Potions (CF 1526C2)​

Soldan sağa iksirlər içirik, canımız heç vaxt mənfi olmamalıdır. Maksimum neçə iksir içmək olar?

Greedy + heap ideyası: hamısını içməyə çalış; can mənfiyə düşəndə indiyə qədər içdiklərindən ən zəhərlisini (ən mənfisini) "geri qaytar":

#include <bits/stdc++.h>
using namespace std;

int main() {
    int n;
    cin >> n;

    priority_queue<long long, vector<long long>, greater<long long>> icilenler;
    long long can = 0;
    for (int i = 0; i < n; i++) {
        long long a;
        cin >> a;
        icilenler.push(a);
        can += a;
        if (can < 0) {
            can -= icilenler.top();   // ən menfi iksiri geri qaytar
            icilenler.pop();
        }
    }
    cout << icilenler.size();
    return 0;
}

"Qərarı ver, sonra ən pis qərarı geri al" — heap-in ən gözəl istifadələrindən biri. 26-cı dərsdə (greedy) bu ideyaya qayıdacağıq.

sort, set, yoxsa priority_queue?​

VəziyyətAlət
Bütün data əldədir, bir dəfə sıralasort
Axtarış + silmə + "x-dən böyük ilk"set / multiset
Yalnız ən böyük/kiçik lazımdır, tez-tez push/poppriority_queue

priority_queue set-dən sürətlidir (sabiti kiçikdir), amma yalnız zirvəni görür — ortadan silmək, axtarmaq olmur.

Çalışmalar​

Çalışma 1. n ədəd axınla gəlir. Hər yeni ədəddən sonra indiyə qədərki 3 ən böyüyün cəmini çap edin (ilk 2 ədəddə mövcud olanların cəmini).

Həlli
#include <bits/stdc++.h>
using namespace std;

int main() {
    int n;
    cin >> n;

    priority_queue<int, vector<int>, greater<int>> pq;  // 3 ən böyük
    long long cem = 0;
    for (int i = 0; i < n; i++) {
        int x;
        cin >> x;
        pq.push(x);
        cem += x;
        if (pq.size() > 3) {
            cem -= pq.top();
            pq.pop();
        }
        cout << cem << '\n';
    }
    return 0;
}

Cəmi ayrıca saxlayırıq ki, hər addımda heap-i boşaltmayaq.

Çalışma 2. Çeşidlənmiş k siyahını bir siyahıya birləşdirin (hamısının ümumi uzunluğu n). O(n log k) həll qurun. İpucu: heap-də hər siyahıdan bir "namizəd".

Həlli
// siyahilar: vector<vector<int>> s (hər biri artan)
priority_queue<
    pair<int, pair<int,int>>,                 // {qiymet, {siyahi, indeks}}
    vector<pair<int, pair<int,int>>>,
    greater<>
> pq;

for (int i = 0; i < k; i++)
    if (!s[i].empty()) pq.push({s[i][0], {i, 0}});

while (!pq.empty()) {
    auto [q, yer] = pq.top(); pq.pop();
    auto [si, ind] = yer;
    cout << q << ' ';
    if (ind + 1 < (int)s[si].size())
        pq.push({s[si][ind + 1], {si, ind + 1}});
}

Hər element heap-ə bir dəfə girib çıxır → O(n log k). Bu, merge sort-un k-yollu ümumiləşməsidir.

Tez-tez edilən səhvlər​

  1. Min-heap yazılışını səhv salmaq — greater<int> üçüncü parametrdir, ikinci yox.
  2. Boş heap-də top()/pop() çağırmaq — undefined behavior; əvvəl empty() yoxlayın.
  3. pq.size() > k müqayisəsində size()-ın unsigned olduğunu unutmaq — (int)pq.size() yazmaq təhlükəsizdir.
  4. Heap-lə ortadan element silməyə çalışmaq — bu iş üçün multiset lazımdır.
  5. Cəm/qiymətlərdə long long unutmaq.
