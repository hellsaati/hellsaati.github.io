Həll saatı 22. Memoization və 1D DP

Keçən dərs fib(n)-in eksponensial böyüdüyünü gördük: eyni alt məsələ dönə-dönə hesablanır. Dinamik proqramlaşdırma (DP) bu israfın dərmanıdır: Hər alt məsələni bir dəfə həll et, cavabını saxla, təkrar soruşulanda hazırından ver. Buna görə də DP olimpiadaların bəlkə də ən vacib mövzusudur.

Memoization: rekursiya + yaddaş​

fib-i xilas edək — cəmi iki sətir əlavə olunur:

#include <bits/stdc++.h>
using namespace std;

long long memo[95];
bool dp[95];

long long fib(int n) {
    if (n <= 1) return n;
    if (dp[n]) return memo[n];      // hazir cavab
    dp[n] = true;
    return memo[n] = fib(n - 1) + fib(n - 2);
}

İndi hər fib(k) bir dəfə hesablanır, Buna görə də O(2ⁿ) yox, O(n) olur.

Memoization ("yaddaşa yazma") üsulunun başqa adı isə top-down DP-dir. Memoization bunlardan ibarətdir:

  1. Rekursiv həlli yaz
  2. Cavabı arqumentlərə görə massivdə saxla
  3. Girişdə "hazırdırsa qaytar" yoxlaması qoy

Bottom-up: cədvəli aşağıdan doldurmaq​

Eyni fikri rekursiyasız yazmaq olar:

long long dp[95];
dp[0] = 0; dp[1] = 1;
for (int i = 2; i <= n; i++)
    dp[i] = dp[i - 1] + dp[i - 2];
cout << dp[n];

DP düşüncə çərçivəsi​

Hər DP məsələsində üç sual verin:

  1. Vəziyyət (state): dp[i] nəyi bildirir? — dəqiq cümlə ilə deyin!
  2. Keçid (transition): dp[i]-ni əvvəlki vəziyyətlərdən necə alırıq?
  3. Base case: ən kiçik hal nədir?

Nümunə 1: Dice Combinations (CSES 1633)​

Zər ataraq (hər atış 1..6) cəmi n etməyin neçə yolu var? (mod 10⁹+7)

  1. Vəziyyət: dp[i] = cəmi düz i etməyin yollarının sayı
  2. Keçid (transition): son atış 1,2,...,6 ola bilər → dp[i] = dp[i-1] + dp[i-2] + ... + dp[i-6]
  3. Base case: dp[0] = 1 (heç nə atmamaq — bir yol)
#include <bits/stdc++.h>
using namespace std;
const long long MOD = 1e9 + 7;

int main() {
    int n;
    cin >> n;

    vector<long long> dp(n + 1, 0);
    dp[0] = 1;
    for (int i = 1; i <= n; i++)
        for (int z = 1; z <= 6 and z <= i; z++)
            dp[i] = (dp[i] + dp[i - z]) % MOD;

    cout << dp[n];
    return 0;
}
Niyə mod?

Yolların sayı astronomik böyüyür; məsələ cavabı 10⁹+7-yə bölünmə qalığı ilə istəyir. Hər toplamadan sonra % MOD alırıq, overflow da olmur. Mod arifmetikasının qaydaları 27-ci dərsdə dərinləşəcək.

Nümunə 2: Minimizing Coins (CSES 1634)​

Qəpiklər c₁..cₖ (istənilən sayda istifadə olunur); cəmi x etmək üçün minimum qəpik sayı?

  1. Mənası: dp[i] = i məbləğini yığmaq üçün minimum qəpik
  2. Transition: son qəpik c ola bilər → dp[i] = min(dp[i - c] + 1) bütün c-lər üzrə
  3. Base Case: dp[0] = 0; mümkünsüz hallar üçün "sonsuzluq"
#include <bits/stdc++.h>
using namespace std;
const long long INF = 1e18;

int main() {
    int n; long long x;
    cin >> n >> x;

    vector<long long> c(n);
    for (auto &v : c) cin >> v;

    vector<long long> dp(x + 1, INF);
    dp[0] = 0;
    for (long long i = 1; i <= x; i++)
        for (long long v : c)
            if (v <= i and dp[i - v] != INF)
                dp[i] = min(dp[i], dp[i - v] + 1);

    cout << (dp[x] == INF ? -1 : dp[x]);
    return 0;
}

Greedy ("həmişə ən böyük sikkəni götür") burada işləmir: c = 4, x = 6 üçün greedy 4+1+1 = 3 qəpik verir, optimal isə 3+3 = 2. DP bütün variantları sistemli yoxladığı üçün belə tələlərə düşmür.

Nümunə 3: Removing Digits (CSES 1637)​

n-dən hər addımda rəqəmlərindən birini çıxırıq; 0-a minimum addım? (27 → 20 → 18 → …)

vector<int> dp(n + 1, INT_MAX);
dp[0] = 0;
for (int i = 1; i <= n; i++)
    for (int t = i; t > 0; t /= 10) {
        int r = t % 10;
        if (r > 0) dp[i] = min(dp[i], dp[i - r] + 1);
    }
cout << dp[n];

Vəziyyət tərifi eynidir — dəyişən yalnız keçiddir. Çərçivəyə oturtmağı öyrənsəniz, yeni DP məsələləri tanış görünməyə başlayacaq.

Yolların sayı vs optimal qiymət​

Yuxarıdakı üç məsələ DP-nin iki əsas tipini göstərir:

  • Sayma: dp[i] = Σ dp[keçidlər] (Dice Combinations) — cavab mod ilə
  • Optimallaşdırma: dp[i] = min/max(dp[keçidlər] + xərc) (Minimizing Coins)

Struktur eynidir, yalnız Σ ↔ min dəyişir. Coin Combinations I — sayma tipində qəpik məsələsidir: özünüz müqayisə edin.

Çalışmalar​

Çalışma 1. Pilləkəndə n pillə; bir addımda 1 və ya 2 pillə qalxmaq olar. Zirvəyə neçə yolla qalxmaq olar? (Cavabı mod almadan, n ≤ 80 üçün.)

Həlli
long long dp[81];
dp[0] = 1; dp[1] = 1;
for (int i = 2; i <= n; i++)
    dp[i] = dp[i - 1] + dp[i - 2];
cout << dp[n];

Bu, Fibonacci-nin özüdür — DP-nin "salam dünya"sı. n = 80 üçün cavab ~2.3 * 10¹⁶ — hələ long long-a sığır.

Çalışma 2. Massiv a₁..aₙ verilir. Qonşu olmayan elementlərdən maksimum cəm seçin ("ev yarma" məsələsi). Vəziyyəti özünüz qurun.

Həlli
// dp[i] = ilk i elementden (qonshu goturmemekle) maksimum cem
// kecid: a[i]-ni goturme (dp[i-1]) ve ya gotur (dp[i-2] + a[i])
vector<long long> dp(n + 1, 0);
dp[1] = max(0LL, a[1]);
for (int i = 2; i <= n; i++)
    dp[i] = max(dp[i - 1], dp[i - 2] + a[i]);
cout << dp[n];

"Götür / götürmə" qərarı DP vəziyyətinə çevrildi — 21-ci dərsin rekursiv gəzintisi ilə eyni qərar ağacı, amma hər vəziyyət bir dəfə.

Çalışma 3. Çalışma 2-ni memoization üslubunda (top-down) yazın.

Həlli
long long memo[200005];
bool hazir[200005];

long long f(int i) {
    if (i <= 0) return 0;
    if (i == 1) return max(0LL, a[1]);
    if (hazir[i]) return memo[i];
    hazir[i] = true;
    return memo[i] = max(f(i - 1), f(i - 2) + a[i]);
}

Rekursiv tərif + yaddaş = eyni O(n). Hansı üslub rahatdırsa, onu işlədin — amma ikisini də oxuya bilməlisiniz.

Tez-tez edilən səhvlər​

  1. dp[0] bazasını səhv qoymaq (sayma məsələsində dp[0] = 1 olur — "boş yol").
  2. Sayma məsələsində mod almağı unutmaq / overflow.
  3. Optimallaşdırmada INF halları yoxlamadan dp[i - c] + 1 yazmaq.
  4. Vəziyyətin dəqiq tərifini yazmadan koda girişmək — ən böyük səhv: əvvəl cümləni deyin, sonra kodu.
  5. Keçiddə i - z < 0 sərhədini yoxlamamaq.
