Həll saatı 16. sort() və comparator-lar

12-ci dərsdə sıralamanı əl ilə qurduq — O(n²). İndi peşəkar alət: STL-in sort() funksiyası — O(n log n), n = 10⁶-nı saniyənin onda birində sıralayır. Bu dərsdən sonra sıralama sizin üçün bir sətirdir; əsl mövzu — nəyə görə sıralamağı təyin etməkdir.

sort() əsasları​

#include <algorithm>

vector<int> a = {5, 2, 8, 1};
sort(a.begin(), a.end());                    // artan: 1 2 5 8
sort(a.begin(), a.end(), greater<int>());    // azalan: 8 5 2 1

int b[100];
sort(b, b + n);                              // adi massiv de olur

sort(başlanğıc, son) — son daxil deyil. Massivdə b + n, vectorda a.end().

Comparator — öz qaydanla sırala​

Comparator — "a b-dən əvvəl gəlməlidirmi?" sualına cavab verən funksiya:

bool cmp(int x, int y) {
    return x % 10 < y % 10;     // son reqeme gore artan
}
sort(a.begin(), a.end(), cmp);

Qaydalar:

  • true = "x mütləq y-dən əvvəl"
  • Bərabərlik halında false qaytarmalıdır (< yazın, <= YOX — <= runtime error verə bilər!)

Müasir üslub — lambda (yerində yazılan funksiya):

sort(a.begin(), a.end(), [](int x, int y) {
    return x > y;               // azalan
});

pair — cütlüklər​

12-ci dərsdə paralel massivləri sinxron dəyişmək əziyyətini çəkdik. Həll — pair:

pair<int, string> p = {85, "Aysel"};
p.first;    // 85
p.second;   // "Aysel"

vector<pair<int, string>> teleber(n);
for (auto &t : teleber) cin >> t.second >> t.first;  // ad bal

sort(teleber.begin(), teleber.end());

pair avtomatik əvvəl first, bərabərlikdə second üzrə müqayisə olunur. "Bala görə art, bərabərdirsə ada görə" = balı first qoy, sırala, bitdi.

Azalan bal + artan ad kimi qarışıq sıra üçün comparator:

sort(t.begin(), t.end(), [](auto &a, auto &b) {
    if (a.first != b.first) return a.first > b.first;  // bal: azalan
    return a.second < b.second;                        // ad: artan
});
İndeksi yadda saxlamaq

Sıralamadan sonra "bu element əvvəl neçənci idi?" lazım olursa: vector<pair<int,int>> b; b.push_back({a[i], i}); — qiymət + ilkin indeks cütünü sıralayın.

Sıralama + greedy: ilk dadım​

Sıralamanın əsl gücü ondan sonra gələn addımdadır. Klassik nümunə — Movie Festival (CSES): n filmin başlama/bitmə vaxtı; maksimum neçə filmə tam baxmaq olar?

Açar fikir: bitmə vaxtına görə sırala, sonra mümkün olan hər filmi götür:

#include <bits/stdc++.h>
using namespace std;

int main() {
    int n;
    cin >> n;
    vector<pair<int, int>> f(n);           // {bitme, bashlama}!
    for (auto &x : f) cin >> x.second >> x.first;

    sort(f.begin(), f.end());              // bitmeye gore

    int say = 0, vaxt = 0;
    for (auto &x : f) {
        if (x.second >= vaxt) {            // bashlaya bilirikse
            say++;
            vaxt = x.first;                // yeni mesgulluq sonu
        }
    }
    cout << say;
    return 0;
}

Niyə bitməyə görə? Ən tez bitən film qalan vaxta ən çox yer saxlayır. Bu düşüncə tərzi — greedy — 26-cı dərsdə ayrıca mövzudur; burada ilk nümunəsini görürsünüz.

{bitme, bashlama} sırasına baxın: pair-in avtomatik müqayisəsindən yararlanmaq üçün sıralama açarını first-ə qoyduq — comparator yazmağa ehtiyac qalmadı.

Sıralamanın stabil qohumu​

stable_sort — bərabər elementlərin ilkin sırasını qoruyur; sort qorumur. "Bala görə sırala, bərabər ballıların giriş sırası saxlansın" tipli tələblərdə stable_sort işlədin. İnterfeys eynidir.

Çalışmalar​

Çalışma 1. n nöqtə (x, y) verilir. Başlanğıcdan (0,0) məsafəyə görə artan sıralayın.

Həlli
sort(p.begin(), p.end(), [](auto &a, auto &b) {
    long long da = (long long)a.first*a.first + (long long)a.second*a.second;
    long long db = (long long)b.first*b.first + (long long)b.second*b.second;
    return da < db;
});

Məsafənin kvadratı ilə müqayisə — sqrt-sız, dəqiq (9-cu dərs).

Çalışma 2. Sətirləri uzunluğa görə, bərabər uzunluqda əlifba sırası ilə düzün.

Həlli
sort(v.begin(), v.end(), [](string &a, string &b) {
    if (a.size() != b.size()) return a.size() < b.size();
    return a < b;
});

Çalışma 3. Ferris Wheel (CSES 1090) həll edin: hər qondolaya maks 2 uşaq, çəki cəmi ≤ x; minimum qondola sayı? İpucu: sırala + iki ucdan.

Həlli
sort(a.begin(), a.end());
int l = 0, r = n - 1, say = 0;
while (l <= r) {
    if (a[l] + a[r] <= x) l++;   // ən yüngul + ən ağır birgə sigirsa
    r--;                          // ağır həmişə gedir
    say++;
}
cout << say;

Ən ağır uşaq mütləq bir qondola tutacaq, əsas sual "yanına ən yüngülü qoşa bilirikmi?"-dir.

Tez-tez edilən səhvlər​

  1. Comparator-da <= yazmaq — bərabər elementlərdə çökə bilər.
  2. sort(a.begin(), a.end) — mötərizə unutmaq (kompilyasiya xətası, amma mesajı qorxuncdur).
  3. Azalan sıra üçün massivi sıralayıb reverse etmək əvəzinə hər yerdə indeks riyaziyyatı ilə əlləşmək.
  4. pair-də açarı second-a qoyub avtomatik müqayisədən məhrum olmaq.
  5. Sıralamanın ilkin sıranı pozduğunu unutmaq (lazımsa indeksi saxlayın / stable_sort).
