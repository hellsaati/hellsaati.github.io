Həll saatı 6. Massivlərlə iş: reverse, shift, silmə, simulyasiya

Bu dərsdə öyrənəcəyik:

  • sürüşdürmə (shift)
  • elementin silinməsi/daxil edilməsi
  • simulyasiya məsələləri

Dövri sürüşdürmə (cyclic shift)​

Massivi 1 mövqe sağa sürüşdürmək: [1,2,3,4,5]: [5,1,2,3,4]

int ls = a[n - 1];
for (int i = n - 1; i >= 1; i--)
    a[i] = a[i - 1];
a[0] = ls;

Diqqət: Sağa sürüşdürmədə dövr axırdan əvvələ gəlir. Əvvəldən getsəydik, a[0] bütün massivə yayılardı.

Sola sürüşdürmə isə əksinədir:

int fr = a[0];
for (int i = 0; i < n - 1; i++)
    a[i] = a[i + 1];
a[n - 1] = fr;
k addım sürüşdürmə

k dəfə tək-tək sürüşdürmək O(n * k)-dır. Daha qısa yazmaq üçün b[(i + k) % n] = a[i] kimi yazmaq olar. T'k gedişə edirik deyə, zaman səmərəliyi O(n) olur.

Elementin silinməsi və daxil edilməsi​

Massivdə 1 elementi "silmək" həmin elementin sağındakıları 1 index sola çəkmək deməkdir:

// "pos" mövqeyindəki elementi sil
for (int i = pos; i < n - 1; i++)
    a[i] = a[i + 1];
n--;

Daxil etmədə isə əksinə edirik:

// "pos" mövqeyinə "x" daxil et
for (int i = n; i > pos; i--)
    a[i] = a[i - 1];
a[pos] = x;
n++;

Hər ikisi O(n)-dir. Bu effektivsizlik səbəbindən tez-tez silmə/əlavə tələb edən məsələlərdə başqa strukturlar (list, set, deque) istifadə olunur, hansiki STL bölməsində görəcəyik. Amma mexanikanı əl ilə bilmək vacibdir.

Simulyasiya məsələləri​

Simulyasiya məsələdə təsvir olunan prosesi kodda addım-addım təkrar etməkdir. Alqoritmik hiylə yoxdur, sadəcə dəqiqlik və səliqə lazımdır.

Nümunə: Queue at the School​

Növbədə oğlanlar (B) və qızlar (G) dayanıb. Hər saniyə, oğlan qızın qarşısındadırsa, yerlərini dəyişirlər. t saniyədən sonra növbə necə görünər?

#include <iostream>
using namespace std;

int main() {
    int n, t;
    string s;
    cin >> n >> t >> s;

    while (t--) {
        for (int i = 0; i + 1 < n; i++) {
            if (s[i] == 'B' and s[i + 1] == 'G') {
                swap(s[i], s[i + 1]);
                i++;   // dəyişən cütü ötür: eyni qız iki dəfə keçməsin
            }
        }
    }
    cout << s;
    return 0;
}

İki incə məqam:

  1. while (t--): "t dəfə təkrarla"-nın qısa yazılışı.
  2. i++ swap-dən sonra dəyişən cütü ötürürük. Bunsuz BGG bir saniyədə GGB olardı (qız iki addım atardı), halbuki düzgün cavab GBG-dir.

Simulyasiyada belə detallar həlledicidir. Kiçik nümunəni kağızda əl ilə yoxlayın, sonra kodun eyni nəticəni verdiyini yoxlayın.

Nümunə: Insomnia cure​

Dörd ədəd k, l, m, n və d verilir: 1-dən d-yə qədər ədədlərdən neçəsi k, l, m, n-dən ən azı birinə bölünür?

int say = 0;
for (int i = 1; i <= d; i++)
    if (i % k == 0 or i % l == 0 or i % m == 0 or i % n == 0)
        say++;
cout << say;

d ≤ 10⁵ olduğundan sadə dövr bəsdir. Hər şeyi hiyləgər düsturla həll etməyə çalışmayın əvvəlcə limitlərə baxın: sadə həll keçirsə, sadə həll yazın.

İki massivin birləşdirilməsi​

Sıralanmış iki massivi bir sıralanmış massivə birləşdirmək:

int i = 0, j = 0, k = 0;
while (i < n and j < m) {
    if (a[i] <= b[j]) c[k++] = a[i++];
    else c[k++] = b[j++];
}
while (i < n) c[k++] = a[i++];
while (j < m) c[k++] = b[j++];

Hər addımda iki "başın" kiçiyini götürürük. Sonda qalan quyruğu köçürürük. Bu 10 sətri əzbər bilin, gələn dərslərdə lazım olacaq.

Çalışmalar​

Çalışma 1. Massivdən bütün sıfırları silin (qalanların sırası saxlanmaqla).

Həlli
int m = 0;
for (int i = 0; i < n; i++)
    if (a[i] != 0)
        a[m++] = a[i];
n = m;

Tək-tək silmək O(n²) olardı. "Two pointers" üsulu ilə bunu bir gedişdə həll etmək olar.

Çalışma 2. Massivi k indeks sola dövri sürüşdürün (bir gedişlə).

Həlli
for (int i = 0; i < n; i++)
    b[i] = a[(i + k) % n];

Çalışma 3. [1,1,2,2,2,3,1,1] kimi massivdə ardıcıl təkrarları silin: 1 2 3 1 çap edin.

Həlli
for (int i = 0; i < n; i++) {
    if (i == 0 or a[i] != a[i - 1])
        cout << a[i] << ' ';
}

Element yalnız qrupun birincisidirsə çap olunur, ona görə, ya i == 0, ya da əvvəlkindən fərqlidir. or-in qısa qapanması burada a[-1]-dən qoruyur.

Tez-tez edilən səhvlər​

  1. Sağa sürüşdürmədə dövrü əvvəldən başlayıb massivi tək qiymətlə doldurmaq;
  2. Silmə zamanı n-- unutmaq;
  3. Simulyasiyada bir addımın daxilində "artıq dəyişmiş" elementi yenidən emal etmək;
  4. while (t--) ilə while (--t) fərqi.
