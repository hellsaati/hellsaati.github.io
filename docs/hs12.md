# Həll saatı 11. Sürətli I/O və çap formatı

Bəzən düzgün alqoritm yazırsınız, amma TLE alırsınız. Bu zaman səhv alqoritm yox, input/output-da olur. 

Bu dərsdə öyrənəcəyik:

- `cin/cout`-u sürətləndirmək
- Kəsr ədədlərin dəqiq çapı (`fixed`, `setprecision`)
- Giriş formatlarının incəlikləri

## Sehrli iki sətir

Hər CP proqramının əvvəlinə yazın:

```cpp
int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(NULL);
    ...
}
```

Nə edir:

- `sync_with_stdio(false)`: C++ axınlarının C-nin `printf/scanf`-i ilə sinxronizasiyasını söndürür.
- `cin.tie(NULL)`: Input-u sürətləndirir.

:::danger[Qarışdırmayın]
Bu sətirlərdən sonra `scanf/printf` istifadə etməyin. Birini seçin: ya hər yerdə `cin/cout`, ya hər yerdə `scanf/printf`.
:::

### endl və '\n'

```cpp
cout << x << endl;    // yeni sətir + buferi boşalt (YAVAŞ)
cout << x << '\n';    // yalnız yeni sətir (SÜRƏTLİ)
```

`endl` hər dəfə buferi boşaldır ki, bu da kodun yavaş işləməsinə səbəb olur. Buna görə də həmişə `'\n'` yazın.

## Kəsr ədədlərin çapı

Standart halda `cout` kəsrləri qısaldır (`3.14159` ➝ `3.14159`, amma `1000000.5` ➝ `1e+06`!). Əgər məsələ "vergüldən sonra 6 rəqəmi" istəyirsə:

```cpp
#include <iomanip>

double x = 3.14159265;
cout << fixed << setprecision(6) << x;   // 3.141593
```

Bir dəfə yazmaq bəsdir, bütün sonrakı çaplar üçün keçərlidir.

```cpp
cout << fixed << setprecision(2);
cout << 5.0 / 3 << '\n';    // 1.67
cout << 10.0 / 4 << '\n';   // 2.50: sondakı sıfır da çap olunur
```

## Giriş formatı incəlikləri

### Sayı bilinməyən giriş

```cpp
int x;
while (cin >> x) {
    // giriş bitənə qədər oxu
}
```

### Simvol-simvol oxuma

```cpp
char c;
cin >> c;            //və ya
cin.get(c);           
```

## Çıxış formatına diqqət

Yoxlama sistemləri çıxışı simvol-simvol müqayisə edir. Bu zaman belə hallarda diqqətli olun:

1. "YES" əvəzinə "Yes".
2. Ədədlər arasında boşluq əvəzinə vergül.
3. Sətir sonunda artıq mətn (Məsələn: "Cavab: 42" yalnız "42" istənirsə səhv olaraq götürülür).
4. Son sətirdə `\n` olub-olmaması.

:::tip[Nümunəni birə-bir kopyalayın]
Məsələnin nümunə çıxışı nə göstərirsə, formatınız tam o olmalıdır. "Təxminən oxşar" format WA (Wrong Answer) deməkdir.
:::

## Nümunə: sürət testi

10⁶ ədədi oxuyub cəmini çap edən proqram:

```cpp
#include <iostream>
using namespace std;

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(NULL);

    int n;
    cin >> n;

    long long cem = 0;
    for (int i = 0; i < n; i++) {
        int x;
        cin >> x;
        cem += x;
    }
    cout << cem << '\n';
    return 0;
}
```

Sehrli sətirlərsiz: ≈0.8–1.5 san. Sətirlərlə: ≈0.1–0.2 san. 

## Çalışmalar

**Çalışma 1.** İki tam ədəd verilir. Bölmə nəticəsini vergüldən sonra dəqiq 3 rəqəmlə çap edin.

<details>
<summary>Həlli</summary>

```cpp
#include <iostream>
#include <iomanip>
using namespace std;

int main() {
    long long a, b;
    cin >> a >> b;
    cout << fixed << setprecision(3) << (double)a / b;
    return 0;
}
```

`(double)` çevirməsi olmasa tam bölmə alınar və `.000` çap olunar.

</details>

**Çalışma 2.** Hər sətrində bir neçə ədəd olan sətirlər verilir (sətir sayı bilinmir). Hər sətrin cəmini çap edin.

<details>
<summary>Həlli</summary>

```cpp
#include <iostream>
#include <sstream>
#include <string>
using namespace std;

int main() {
    string setir;
    while (getline(cin, setir)) {
        stringstream ss(setir);
        long long cem = 0, x;
        while (ss >> x) cem += x;
        cout << cem << '\n';
    }
    return 0;
}
```

</details>

## Tez-tez edilən səhvlər

1. `endl`-lə milyon sətir çap edib TLE almaq.
2. Sehrli sətirləri yazıb sonra `scanf`lə qarışdırmaq.
3. `setprecision`-u `fixed`-siz istifadə etmək.
4. Çıxışa artıq mətn əlavə etmək.
5. `cin >> n`-dən sonra `getline` işlətmək.
