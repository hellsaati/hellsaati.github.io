# Həll saatı 23. Prefix sums

Massiv verilir və q dəfə soruşulur: "a..b intervalının cəmi neçədir?" Hər sorğuda dövrlə cəmləmək O(n * q) — n = q = 2 * 10⁵ üçün 4 * 10¹⁰ əməliyyat, ümidsiz. **Prefix cəmləri** hər sorğunu **O(1)**-ə salır. Bir hazırlıq, sonsuz ucuz sorğu — CP-nin ən sevimli barterlərindən.

## İdeya

`pre[i]` = ilk i elementin cəmi (`pre[0] = 0`):

```cpp
vector<long long> pre(n + 1, 0);
for (int i = 1; i <= n; i++)
    pre[i] = pre[i - 1] + a[i];      // a 1-indeksli
```

İstənilən intervalın cəmi iki prefiksin fərqidir:

```
cem(l..r) = pre[r] - pre[l - 1]
```

Niyə: `pre[r]` 1..r-i, `pre[l-1]` 1..l-1-i cəmləyir; fərq — düz l..r.

```
a:      3   1   4   1   5
pre: 0  3   4   8   9   14

cem(2..4) = pre[4] - pre[1] = 9 - 3 = 6   (1+4+1 ✓)
```

### CSES 1646 — tam həll

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n, q;
    cin >> n >> q;

    vector<long long> pre(n + 1, 0);
    for (int i = 1; i <= n; i++) {
        long long x;
        cin >> x;
        pre[i] = pre[i - 1] + x;
    }

    while (q--) {
        int l, r;
        cin >> l >> r;
        cout << pre[r] - pre[l - 1] << '\n';
    }
    return 0;
}
```

:::danger[Təhlükə]
1. Cəmlər `long long`-da — 2 * 10⁵ ədəd * 10⁹ = 2 * 10¹⁴.
2. 1-indeksləmə + `pre[0] = 0` — `l = 1` halı öz-özünə düzgün işləyir. 0-indekslə qarışdırsanız, `l-1 = -1` faciəsi hazırdır.
:::

## Say, orta, cüt-tək: cəmdən o yana

Prefix texnikası yalnız cəm üçün deyil — **istənilən "toplana bilən" şey** üçün işləyir:

```cpp
// İntervalda neçə cüt eded var?
pre[i] = pre[i - 1] + (a[i] % 2 == 0);

// İntervalda 'x' hər neçə dəfə rast gəlinir?
pre[i] = pre[i - 1] + (s[i] == 'x');
```

"İntervalda neçə dənə ...?" sualı görəndə refleks: prefix say massivi.

## 2D prefix: matrisdə düzbucaqlı cəmi

10-cu dərsin 2D massivləri + eyni ideya. `pre[i][j]` = sol-yuxarı (1,1)–(i,j) düzbucaqlısının cəmi:

```cpp
pre[i][j] = a[i][j] + pre[i-1][j] + pre[i][j-1] - pre[i-1][j-1];
```

`pre[i-1][j]` və `pre[i][j-1]` üst-üstə düşən `pre[i-1][j-1]` sahəsini iki dəfə sayır — bir dəfə çıxırıq. Bu, **daxil-et-xaric-et** (inclusion-exclusion) prinsipidir.

Sorğu — (x1,y1)–(x2,y2) düzbucaqlısı:

```cpp
cem = pre[x2][y2] - pre[x1-1][y2] - pre[x2][y1-1] + pre[x1-1][y1-1];
```

### Nümunə: Forest Queries (CSES 1652)

Meşə xəritəsində düzbucaqlıda neçə ağac ('*') var:

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n, q;
    cin >> n >> q;

    vector<vector<int>> pre(n + 1, vector<int>(n + 1, 0));
    for (int i = 1; i <= n; i++) {
        string s;
        cin >> s;
        for (int j = 1; j <= n; j++)
            pre[i][j] = (s[j-1] == '*') + pre[i-1][j] + pre[i][j-1] - pre[i-1][j-1];
    }

    while (q--) {
        int x1, y1, x2, y2;
        cin >> x1 >> y1 >> x2 >> y2;
        cout << pre[x2][y2] - pre[x1-1][y2] - pre[x2][y1-1] + pre[x1-1][y1-1] << '\n';
    }
    return 0;
}
```

## Prefix + map: cəmi x olan subarray sayı

Daha dərin hiylə (CSES 1660): cəmi x olan **ardıcıl** hissələrin sayı. Açar müşahidə:

```
cem(l..r) = x   ⟺   pre[r] - pre[l-1] = x   ⟺   pre[l-1] = pre[r] - x
```

Sağ ucu gəzirik; hər `pre[r]` üçün soruşuruq: "əvvəllər neçə dəfə `pre[r] - x` qiyməti görünüb?" — 17-ci dərsin `map`-i:

```cpp
map<long long, long long> say;
say[0] = 1;
long long pre = 0, cavab = 0;
for (int i = 0; i < n; i++) {
    pre += a[i];
    cavab += say[pre - x];
    say[pre]++;
}
cout << cavab;
```

"Cütü axtarma — tamamlayıcını axtar" (17-ci dərs) + prefix cəmi = O(n log n). Bu kombinasiya yarışlarda çox tez-tez çıxır.

## Kadane: maksimum subarray cəmi

Qohum klassika (CSES 1643): ən böyük cəmli ardıcıl hissə. Prefikslə düşün: `pre[r] - min(pre[0..r-1])` maksimallaşmalıdır:

```cpp
long long pre = 0, minPre = 0, cavab = LLONG_MIN;
for (int i = 0; i < n; i++) {
    pre += a[i];
    cavab = max(cavab, pre - minPre);
    minPre = min(minPre, pre);
}
cout << cavab;
```

Eyni nəticəni verən məşhur **Kadane** yazılışı — "mənfiyə düşən cəmi at":

```cpp
long long cari = 0, cavab = LLONG_MIN;
for (int i = 0; i < n; i++) {
    cari = max((long long)a[i], cari + a[i]);
    cavab = max(cavab, cari);
}
```

İki baxış, bir alqoritm. Hamısı mənfi olan massivdə cavabın ən böyük (ən az mənfi) element olduğuna diqqət — `cavab`-ı 0-dan yox, `LLONG_MIN`-dən başladın.

## Fərq massivi (difference array)

Prefiksin tərsi: q dəfə "l..r intervalına d əlavə et" əməliyyatı, sonda massivi çap et. Hər əməliyyatı O(1) etmək:

```cpp
vector<long long> d(n + 2, 0);
// "l..r-e d elave et":
d[l] += d_qiymet;
d[r + 1] -= d_qiymet;

// sonda berpa — prefix cemi:
for (int i = 1; i <= n; i++) d[i] += d[i - 1];   // d[i] indi a[i]-dir
```

Məntiq: `d`-nin prefiks cəmi hər nöqtədə "aktiv" əlavələrin cəmini verir. İnterval-yeniləmə məsələlərinin açarıdır.

## Çalışmalar

**Çalışma 1.** n ədəd və q sorğu: intervalda cüt ədədlərin sayı.

<summary>Həlli</summary>

```cpp
vector<int> pre(n + 1, 0);
for (int i = 1; i <= n; i++)
    pre[i] = pre[i - 1] + (a[i] % 2 == 0);
// sorgu: pre[r] - pre[l-1]
```

</details>

**Çalışma 2.** Massivdə cəmi 0 olan subarray varmı? O(n log n). İpucu: iki bərabər prefiks.

<details>
<summary>Həlli</summary>

```cpp
set<long long> gorulen = {0};
long long pre = 0;
bool var = false;
for (int i = 0; i < n; i++) {
    pre += a[i];
    if (gorulen.count(pre)) var = true;   // pre[j] == pre[i] → cem(j+1..i) = 0
    gorulen.insert(pre);
}
```

`pre[r] = pre[l-1]` ⟺ aradakı cəm 0. Subarray Sums ideyasının x = 0 halı.

</details>

**Çalışma 3.** n günlük təqvimdə q tədbir: hər biri [l, r] günlərini tutur. Hər gün neçə tədbir var? (Fərq massivi ilə.)

<details>
<summary>Həlli</summary>

```cpp
vector<int> d(n + 2, 0);
while (q--) {
    int l, r;
    cin >> l >> r;
    d[l]++; d[r + 1]--;
}
for (int i = 1; i <= n; i++) {
    d[i] += d[i - 1];
    cout << d[i] << ' ';
}
```

</details>

## Tez-tez edilən səhvlər

1. `int`-də cəm daşması — prefiks massivi həmişə `long long`.
2. 0/1-indeks qarışığı: `pre[r] - pre[l-1]` yalnız 1-indekslə təmizdir.
3. 2D-də `pre[i-1][j-1]`-i geri əlavə etməyi unutmaq.
4. Subarray sayında `say[0] = 1` başlanğıcını unutmaq (l = 1-dən başlayan intervallar itir).
5. Fərq massivində `d[r+1]` üçün yer ayırmamaq (ölçü n+2).
