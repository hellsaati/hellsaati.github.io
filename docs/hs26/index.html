Həll saatı 25. Həndəsə: intervallar, düzbucaqlılar

Olimpiada həndəsəsinin giriş qapısı üçbucaqlar deyil — oxlara paralel intervallar və düzbucaqlılardır. Bu məsələlər düstur yox, diqqətli hal təhlili istəyir. Bu dərsdə koordinat müstəvisində ən çox çıxan üç motivi öyrənirik: interval kəsişməsi, düzbucaqlı kəsişməsi, hadisə üsulu.

İntervalların kəsişməsi​

[a₁, b₁] və [a₂, b₂] intervalları kəsişirmi? Kəsişmənin özü nədir?

long long sol = max(a1, a2);
long long sag = min(b1, b2);

if (sol <= sag) {
    // kesishirler; kesishme [sol, sag], uzunlugu sag - sol
} else {
    // kesishmirler
}

Bu iki sətri əzbər bilin: kəsişmə = [maksimum başlanğıc, minimum son]. Boşdursa (sol > sag), kəsişmə yoxdur.

Açıq, yoxsa qapalı?

"Toxunma sayılır, ya yox?" — məsələdən asılıdır. Nöqtələr tam ədədlərdirsə və toxunma sayılmırsa, şərt sol < sag olur. Məsələ mətnindəki "touch", "overlap", "strictly inside" sözlərinə diqqətlə baxın — bu, WA-ların əsas mənbəyidir.

Düzbucaqlıların kəsişməsi​

Oxlara paralel düzbucaqlı = X intervalı * Y intervalı. Ona görə iki düzbucaqlının kəsişməsi = hər oxda ayrıca interval kəsişməsi:

// duzbucaqli: (x1, y1) sol-ashagi, (x2, y2) sag-yuxari
long long enX = min(ax2, bx2) - max(ax1, bx1);   // X oxunda kesishme uzunlugu
long long enY = min(ay2, by2) - max(ay1, by1);   // Y oxunda

if (enX > 0 and enY > 0) {
    long long sahe = enX * enY;      // kesishme sahesi
} else {
    // kesishmirler (enX <= 0 ve ya enY <= 0)
}

2D məsələni iki müstəqil 1D məsələyə parçalamaq — bu dərsin ən vacib düşüncə tərzidir.

Nümunə: iki düzbucaqlının birləşmə sahəsi​

long long s1 = (ax2 - ax1) * (ay2 - ay1);
long long s2 = (bx2 - bx1) * (by2 - by1);
long long kesishme = max(0LL, min(ax2,bx2) - max(ax1,bx1)) *
                     max(0LL, min(ay2,by2) - max(ay1,by1));
cout << s1 + s2 - kesishme;

Yenə 23-cü dərsin daxil-et-xaric-et prinsipi: iki dəfə sayılanı bir dəfə çıx. max(0LL, ...) — kəsişməyən halda mənfi "uzunluğu" sıfırlayır; səliqəli hiylədir.

Nöqtə düzbucaqlının içindədirmi?​

bool icinde = (x1 <= px and px <= x2 and y1 <= py and py <= y2);

Sərhəd daxildirsə <=, deyilsə <. Sadədir, amma məsələlər bunu 4-5 düzbucaqlı ilə kombinə edəndə hal təhlili tələb olunur — səbrli olun, hər şərti ayrıca yazın.

Hadisə üsulu (sweep line-ın sadə forması)​

"Zamanda intervallar" məsələlərinin açarı: intervalı iki hadisəyə parçala — başlanğıc (+1) və son (−1), hadisələri sırala, sayğacla gəz.

Nümunə: Restaurant Customers (CSES 1619)​

n müştəri; hər biri [gəlmə, getmə] vaxtında içəridədir. Eyni anda maksimum neçə müştəri olub?

#include <bits/stdc++.h>
using namespace std;

int main() {
    int n;
    cin >> n;

    vector<pair<int,int>> hadise;          // {vaxt, +1/-1}
    for (int i = 0; i < n; i++) {
        int a, b;
        cin >> a >> b;
        hadise.push_back({a, +1});
        hadise.push_back({b, -1});
    }
    sort(hadise.begin(), hadise.end());

    int cari = 0, cavab = 0;
    for (auto [vaxt, delta] : hadise) {
        cari += delta;
        cavab = max(cavab, cari);
    }
    cout << cavab;
    return 0;
}

İncə məqam: eyni vaxtda gəlmə və getmə varsa, sıralamada -1 əvvəl gəlir (pair müqayisəsində {t,-1} < {t,+1}), yəni "getmə" əvvəl işlənir. CSES-də vaxtlar fərqli olduğundan problem çıxmır, amma başqa məsələdə bu sıra cavabı dəyişə bilər: məsələnin tərifinə görə seçin.

Bu, 23-cü dərsdəki fərq massivinin "sıralanmış hadisələr" versiyasıdır: koordinatlar böyük (10⁹) olanda massiv yerinə hadisə siyahısı işləyir.

Manhattan məsafəsi​

Şəbəkə şəhərində iki nöqtə arasındakı yol:

long long d = llabs(x1 - x2) + llabs(y1 - y2);

Evklid məsafəsindən (düz xətt) fərqli olaraq kökalma yoxdur, tam ədədlərdə qalır. CP-də "distance" görəndə hansının nəzərdə tutulduğunu yoxlayın; Manhattan çox vaxt hiyləli sadələşmələr verir (məsələn, X və Y müstəqil optimallaşır).

Kökdən qaçın

Evklid məsafələrini müqayisə etmək lazımdırsa, kvadratlarını müqayisə edin: dx*dx + dy*dy. Belə yazmaq daha məntiqlidir, çünki sqrt həm yavaşdır, həm də dəqiqlik itirir. Kök yalnız cavabı çap edəndə alınır.

Çalışmalar​

Çalışma 1. İki interval [a₁,b₁], [a₂,b₂] verilir. Kəsişmə, birləşmə (əgər fasiləsizdirsə) və aradakı boşluğun uzunluğunu çap edin.

Həlli
long long sol = max(a1, a2), sag = min(b1, b2);
if (sol <= sag) {
    cout << "Kesishme: " << sag - sol << '\n';
    cout << "Birleshme: " << max(b1,b2) - min(a1,a2) << '\n';
} else {
    cout << "Kesishmir, boshluq: " << sol - sag << '\n';
}

Kəsişməyəndə sol - sag məhz aradakı məsafəni göstərir.

Çalışma 2. n düzbucaqlı verilir (n ≤ 1000). Hamısının ortaq kəsişməsi varmı, varsa sahəsi nə qədərdir?

Həlli
long long X1 = -4e18, Y1 = -4e18, X2 = 4e18, Y2 = 4e18;
for (int i = 0; i < n; i++) {
    long long x1, y1, x2, y2;
    cin >> x1 >> y1 >> x2 >> y2;
    X1 = max(X1, x1); Y1 = max(Y1, y1);
    X2 = min(X2, x2); Y2 = min(Y2, y2);
}
if (X1 < X2 and Y1 < Y2) cout << (X2 - X1) * (Y2 - Y1);
else cout << 0;

Çalışma 3. n tədbir [başlama, bitmə] verilir. Bir zal var; maksimum neçə tədbirə tam baxmaq olar? (Movie Festival)

Həlli
// Bitmə vaxtına görə sırala, ən tez bitənə üstünlük ver
sort(t.begin(), t.end(), [](auto &a, auto &b){ return a.second < b.second; });

int say = 0;
long long serbest = LLONG_MIN;
for (auto [bash, son] : t)
    if (bash >= serbest) { say++; serbest = son; }
cout << say;

Ən tez bitəni götürmək niyə optimaldır? Növbəti dərsin (greedy) əsas sualı məhz budur. Bu məsələ iki dərsin körpüsüdür.

Tez-tez edilən səhvlər​

  1. Kəsişmə şərtində <= / < qarışdırmaq (toxunma sayılır, ya yox?);
  2. Sahə hesabında int-də overflow problemi;
  3. X və Y oxlarını qarışdırmaq / (x1,y1)-in hansı künc olduğunu yoxlamamaq;
  4. Eyni zamanda baş verən eventlərin sırasını düşünməmək;
  5. sqrt ilə məsafə müqayisəsi.
