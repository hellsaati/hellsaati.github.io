# Həll saatı 26. Greedy alqoritmlər

Greedy (acgöz) alqoritmlər hər addımda indiki vəziyyətə görə ən yaxşı görünən seçimi etməkdir.

Əvvəlki dərslərdə görmüşdük ki, qəpik məsələsində greedy {1,3,4} çoxluğunda uğursuz olur. Bəs nə vaxt uğurlu olur?

## Greedy nə vaxt işləyir?

Greedy həlli iki halda işlətmək olar:

- Greedy həllin düzgünlüyünü isbat edə biliriksə;
- Greedy hissə özündən sonrakı hissələrə təsir etmirsə.

Greedy-ni isbat etmədən yazmaq böyük risk daşıyır. Yarışda ən azı "niyə bu seçim heç nəyi korlamır?" cümləsini özünüzə deyə bilməlisiniz. Deyə bilmirsinizsə, kiçik testlərdə brute force edərək düzgün işlədiyini yoxlamalısınız.

## Nümunə: Movie Festival

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    int n;
    cin >> n;

    vector<pair<long long,long long>> f(n);
    for (auto &[x, y] : f) cin >> x >> y;
    sort(f.begin(), f.end());

    int say = 0;
    long long s = -1e9;
    for (auto [x, y] : f)
        if (x >= s) { say++; s = son; }
    cout << say;
    return 0;
}
```

Niyə "ən tez bitən"?

Deyək ki, optimal həllin ilk filmi f olsun, greedy-nin seçdiyi g isə hamıdan tez bitir. f-i g ilə əvəzlə: g daha tez bitdiyi üçün qalan filmlər yenə də sığır. Deməli greedy ilk seçimi optimala çevrilə bilir, induksiya ilə davamını da isbat etmək olur.

:::note[Fikirləş]
"Ən tez başlayanı götür" və ya "ən qısanı götür" isə işləmir, hər ikisinə qarşı-nümunə qurun.
:::

## Nümunə 2: Ferris Wheel

Hər qondolaya ≤ 2 uşaq minə bilər çəki limiti x-dir. Lazım olan minimum qondola sayını tapın.

Greedy həll olaraq ən ağırı ən yüngüllə cütləşdirməyə çalışa bilərik:

```cpp
sort(a.begin(), a.end());
int sol = 0, sag = n - 1, qondola = 0;
while (sol <= sag) {
    if (sol < sag and a[sol] + a[sag] <= x) sol++;
    sag--;
    qondola++;
}
cout << qondola;
```

İdeya budur ki, ən ağır uşaq onsuz da bir qondola tutacaq. Onunla kimisə mindirmək olursa, ən yüngülü mindirmək heç nəyi korlamır (yüngül başqası ilə də sığardı, amma ağırla sığmaq şansı ən azdır). 

## Nümunə 3: Tasks and Deadlines

`i`-ci iş d[i] müddət çəkir və son tarixi t[i]-dir. Qazanc isə `t[i] - bitmə_vaxtı` olur, hansıki mənfi ola bilər. İşləri optimal sıra ilə görsək, maksimum qazanc nə qədər olar?

Cavab gözlənilməz dərəcədə sadədir: **ən qısa işdən başla** (son tarixlər heç nəyi dəyişmir!):

```cpp
sort(v.begin(), v.end());
long long now = 0, cv = 0;
for (auto [d, t] : v) {
    now += d;
    cv += t - now;
}
```

Niyə? Cəm = Σt[i] − Σbitmə[i]. Burada birinci hissə sabitdir, ikincidə hər işin özündən sonra gələn işləri də öz müddəti qədər yavaşladır deyə, hər işin uzunluğunu özündən sonra gələn işlərin sayına vururuq. Buna görə də, uzunları sola saxlamaq daha məntiqli olur.

## Nümunə 4: Stick Lengths

Bütün çubuqları eyni uzunluğa gətirmək tələb olunur. Bir çubuğun uzunluğunu x qədər dəyişmək üçün x vahid pul ödəməliyik. Hansı uzunluq ən az xərcə səbəb olur?

Kağızda nümunələrə baxsanız, bu uzunluğun median olduğunu görə bilərsiniz:

```cpp
sort(a.begin(), a.end());
long long g = a[n / 2], cem = 0;
for (long long x : a) cem += llabs(x - g);
cout << cem;
```

Niyə ədədi orta yox, median? Hədəfi mediandan 1 vahid tərpətsək: yarıdan çoxu uzaqlaşır, yarıdan azı yaxınlaşır, beləliklə cəm artır. |fərq| cəmində həmişə median qalib gəlir, lakin kvadratik fərqlərdə isə ədədi orta daha optimaldır. Bu faktı yadda saxlayın, qarşınıza dəfələrlə çıxa bilər.

## Greedy + heap

"Potions" məsələsini xatırlayın: "hamısını götür, pisləşəndə ən pisini geri qaytar". Bu da greedy-dir, sadəcə greedy-nin peşmanlıq (regret) greedy adlanan növüdür: qərar geri alına bilən formada saxlanır. Heap "ən pis qərarı" O(log n)-ə verir.

## Greedy, yoxsa DP?

Əmin deyilsinizsə və limitlər imkan verirsə, DP yazın. Greedy hipotezini isə əvvəlki dərslərdə öyrəndiyimiz brute force ilə stress-testdən keçirin. Bu taktika yarışlarda çox kömək olur.

## Çalışmalar

**Çalışma 1.** Movie Festival üçün "ən tez başlayanı götür" və "ən qısanı götür" strategiyalarına qarşı-nümunə qurun.

<details>
<summary>Həlli</summary>

"Ən tez başlayan": [1,100], [2,3], [4,5]. Greedy 1 film götürür, optimal 2.

"Ən qısa": [1,4], [3,5], [4,7]. Ən qısa [3,5] ikisini də bloklayır; optimal [1,4] + [4,7] = 2 film (bağlı/açıq uclara görə bərabərlik qaydasına baxın).

Qarşı-nümunə qurmaq bacarığı greedy-nin yarısıdır: strategiyanı sındıra bilmirsinizsə, inam artır.

</details>

**Çalışma 2.** n müştəri növbədədir; i-cisinin xidmət müddəti t[i]. Növbəni elə düzün ki, gözləmə vaxtlarının cəmi minimum olsun. Strategiya + arqument.

<details>
<summary>Həlli</summary>

Ən qısa xidmət birinci (SJF). Tasks and Deadlines-dəki eyni hesab: i-ci müştərinin müddəti özündən sonrakıların hamısının gözləməsinə girir → qısalar qabağa. İki qonşunu yerdəyişdirmə fərqi yenə `d_qabaq − d_sonra`.

```cpp
sort(t.begin(), t.end());
long long gozleme = 0, cari = 0;
for (int i = 0; i < n - 1; i++) {   // sonuncunun gozlemesine ehtiyac yoxdur
    cari += t[i];
    gozleme += cari;
}
```

</details>

**Çalışma 3.** Qəpiklər {1, 2, 5, 10, 20, 50, 100} (AZN qəpikləri kimi). x məbləğini minimum qəpik ilə yığın.

<details>
<summary>Həlli</summary>

```cpp
int coins[] = {100, 50, 20, 10, 5, 2, 1};
int say = 0;
for (int s : coins) { say += x / s; x %= s; }
cout << say;
```

Bu çoxluq "kanonik"dir: hər qəpik özündən kiçiklərin elə kombinasiyasıdır ki, böyüyü götürməmək heç vaxt qazandırmır. {1,3,4}-də isə 3+3 = 6 < 4+1+1 olur və kanoniklik pozulur. Ümumi halda qəpik məsələsi DP-dir; greedy isə yalnız xüsusi qəpiklərdə alınır, hansı ki kanoniklik qorunur.

</details>

## Tez-tez edilən səhvlər

1. İsbatsız greedy yazıb WA almaq;
2. Səhv sıralama açarı (başlama vaxtı ≠ bitmə vaxtı!);
3. Bərabərlik hallarını (eyni bitmə vaxtı, toxunan intervallar) diqqətsiz işləmək;
4. Greedy işləməyəndə israr etmək;
5. `long long` (gözləmə cəmləri, mükafat cəmləri sürətlə böyüyür).
