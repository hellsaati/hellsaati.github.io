Həll saatı 2. Dövrlər: for və while

Kompüterin əsl gücü eyni əməliyyatı milyonlarla dəfə yorulmadan təkrarlamasındadır. Bunu dövrlər (loops) edir.

Bu dərsdə öyrənəcəyik:

  • for və while dövrləri
  • break və continue əmrləri
  • Sayğac, cəm, maksimum kimi ən çox rastlanan dövrlər

while dövrü​

while "nə qədər ki şərt doğrudur, təkrarla" deməkdir:

#include <iostream>
using namespace std;

int main() {
    int i = 1;
    while (i <= 5) {
        cout << i << ' ';
        i++;
    }
    // Çıxış: 1 2 3 4 5
    return 0;
}

Sonsuz dövr

i++ sətrini unutsanız, şərt heç vaxt yanlış olmur və proqram donub qalır. Olimpiadada buna TLE (Time Limit Exceeded) deyilən nəticə uyğun gəlir. Dövr yazanda həmişə öz-özünüzdən soruşun: "bu dövr nə vaxtsa bitəcəkmi?"

for dövrü​

for sayğaclı dövrlər üçün qısa formadır. Üç hissədən ibarətdir: başlanğıc; şərt; addım.

for (int i = 1; i <= 5; i++) {
    cout << i << ' ';
}

Bu kod yuxarıdakı while ilə tam eynidir. Hansını nə vaxt işlətməliyik?

Əgər təkrar sayını əvvəlcədən biliriksə, for, əks halda while dövrünü işlətmək daha rahat və məqsədəuyğundur.

Bir neçə faydalı variant:

for (int i = 0; i < n; i++)      // 0-dan n-ə qədər (n daxil deyil; ən çox istifadə olunan)
for (int i = n; i >= 1; i--)     // geriyə doğru: n, n-1, ..., 1
for (int i = 0; i < n; i += 2)   // 2 addımla: 0, 2, 4, ...
Niyə 0-dan başlayırıq?

Növbəti dərslərdə görəcəyik ki, massivlərdə nömrələmə 0-dan başlayır. Ona görə CP-də ən standart dövr for (int i = 0; i < n; i++) formasıdır.

Klassik tiplər​

1. Cəm​

int n;
cin >> n;

int cem = 0;
for (int i = 1; i <= n; i++) {
    int x;
    cin >> x;
    cem += x;
}
cout << cem;

cem dəyişənini 0-dan başlatmağı unutmayın.

2. Sayğac (neçə dənə?)​

Girişdə verilən n ədəddən neçə dənəsi cütdür?

int say = 0;
for (int i = 0; i < n; i++) {
    int x;
    cin >> x;
    if (x % 2 == 0) say++;
}
cout << say << "sayda ədəd cütdür.";

3. Maksimum / minimum​

Girişdə verilən n ədəddən ən böyüyü hansıdır?

int maks = -1000000000;
for (int i = 0; i < n; i++) {
    int x;
    cin >> x;
    if (x > maks) maks = x;
}
cout << maks;

Başlanğıc qiyməti mümkün qiymətlərin hamısından kiçik olmalıdır.

4. Yoxlama bayrağı (flag)​

bool tapildi = false;
for (int i = 0; i < n; i++) {
    int x;
    cin >> x;
    if (x == 7) tapildi = true;
}
if (tapildi) cout << "7 var.";
else cout << "7 yoxdur.";

break və continue​

  • break dövrü dərhal tamamilə dayandırır
  • continue cari addımı ötürüb növbəti addıma keçir

Nümunə 1: Girişdə verilən n ədəddən ilk mənfi olanı tap.

for (int i = 0; i < n; i++) {
    int x;
    cin >> x;
    if (x < 0) {
        cout << "İlk mənfi: " << x;
        break;
    }
}

Nümunə 2: Girişdə verilən n ədəddən yalnız müsbət olanlarının cəmini tap.

int cem = 0;
for (int i = 0; i < n; i++) {
    int x;
    cin >> x;
    if (x <= 0) continue;
    cem += x;
}
Qeyd

break yalnız bir dövrü dayandırır. İç-içə dövrlərdə xarici dövr davam edir, buna növbəti dərsdə qayıdacağıq.

Nümunə:​

Collatz ardıcıllığı (CSES 1068)

Məşhur "Weird Algorithm": n-dən başlayırıq; cütdürsə 2-yə bölürük, təkdirsə 3-ə vurub 1 əlavə edirik; 1-ə çatanda dayanırıq.

#include <iostream>
using namespace std;

int main() {
    long long n;
    cin >> n;

    cout << n;
    while (n != 1) {
        if (n % 2 == 0) n = n / 2;
        else n = 3 * n + 1;
        cout << ' ' << n;
    }
    return 0;
}

Burada iki incə məqam var:

  1. long long: n 10⁶ olsa belə, aralıq qiymətlər int-in yuxarı sərhədini (təxminən 2 * 10⁹) aşa bilər.
  2. Çap formatı: ilk ədəddən əvvəl boşluq olmamalıdır. Ona görə n-i əvvəlcədən çap edib sonra hər yeni qiyməti ' ' << n şəklində yazırıq.

Çalışmalar​

Çalışma 1. n verilir. 1-dən n-ə qədər ədədlərdən 3-ə bölünənlərin cəmini tapın.

Həlli
#include <iostream>
using namespace std;

int main() {
    int n;
    cin >> n;

    int cem = 0;
    for (int i = 3; i <= n; i += 3) cem += i;
    cout << cem;
    return 0;
}

Diqqət: i += 3 addımı ilə yalnız 3-ə bölünənləri gəzirik. Bu, hər ədədi yoxlamaqdan 3 dəfə sürətlidir.

Çalışma 2. n ədəd verilir. Onların neçəsinin özündən əvvəlkindən böyük olduğunu tapın (birinci ədəd sayılmır).

Həlli
#include <iostream>
using namespace std;

int main() {
    int n;
    cin >> n;

    int evvelki, say = 0;
    cin >> evvelki;
    for (int i = 2; i <= n; i++) {
        int x;
        cin >> x;
        if (x > evvelki) say++;
        evvelki = x;
    }
    cout << say;
    return 0;
}

Bütün ədədləri yadda saxlamağa ehtiyac yoxdur, yalnız əvvəlkini saxlamaq bəsdir. Massivsiz işləyən bu üslub yaddaş baxımından çox səmərəlidir.

Çalışma 3. n!-ı hesablayan kod yazın (n <= 20 üçün). Hansı tip lazımdır?

Həlli
#include <iostream>
using namespace std;

int main() {
    int n;
    cin >> n;

    long long f = 1;
    for (int i = 2; i <= n; i++) f *= i;
    cout << f;
    return 0;
}

20! ≈ 2.4 * 10¹⁸ long long-un sərhədinə (≈9.2 * 10¹⁸) sığır, amma 21! artıq sığmır. Tip sərhədlərini bilmək CP-də həyati vacibdir.

Tez-tez edilən səhvlər​

  1. Sayğacı artırmağı unudub sonsuz dövrə düşmək;
  2. for (int i = 0; i <= n; i++): < əvəzinə <= yazıb bir addım artıq getmək (off-by-one);
  3. Cəmi int-də saxlayıb daşma (overflow) almaq;
  4. Maksimum axtarışında başlanğıc qiyməti düzgün seçməmək;
  5. Dövr daxilində cin oxumağı unutmaq.
