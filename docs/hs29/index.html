Həll saatı 28. Hashing: unordered set və map

Kampın son həll saatına xoş gəldiniz. Əvvəlki dərslərdə set və map-in O(log n) olduğunu gördük. Hash cədvəli eyni əməliyyatları normal halda O(1)-ə edir. Bu dərsdə unordered_set və unordered_map-in necə işlədiyini, nə vaxt seçilməli olduğunu və string hashing-in əsas ideyasını öyrənirik.

Hash cədvəli necə işləyir?​

İdeya: elementi hash funksiyası ilə ədədə çevir, o ədədlə massivdə "yuva" (bucket) seç və elementi orada saxla:

x → hash(x) → yuva nomresi = hash(x) % yuva_sayi

Toqquşma (collision): iki fərqli element eyni yuvaya düşə bilər. Pis halda hamısı bir yuvaya düşür, buna görə də əməliyyatlar O(n)-ə qədər yavaşıyır. Buna görə deyirik: ortalamada O(1), ən pis halda isə O(n) ola bilər.

unordered_set / unordered_map​

İstifadəsi adi set/map ilə, demək olar ki, eynidir:

#include <unordered_set>
#include <unordered_map>

unordered_set<int> s;
s.insert(5);
s.count(5);
s.erase(5);

unordered_map<string, int> say;
say["salam"]++;
set və map-ə görə nə itiririk?

Elementlər ixtiyari ardıcıllıqla yığıla bilər. Buna görə də lower_bound və upper_bound işləmir.

Nümunə məsələlər​

// Distinct Numbers: unordered_set ilə
unordered_set<int> s;
for (int i = 0; i < n; i++) { cin >> x; s.insert(x); }
cout << s.size();

// Sum of Two Values: unordered_map ilə:
unordered_map<int, int> yer;
for (int i = 1; i <= n; i++) {
    cin >> a;
    if (yer.count(x - a)) { cout << yer[x - a] << ' ' << i; return 0; }
    yer[a] = i;
}
cout << "IMPOSSIBLE";

Kod map versiyası ilə hərfbəhərf eynidir, yalnız unordered versiyası ilə əvəz olunub.

Codeforces-da anti-hash testlər

Codeforces-da iştirakçılar unordered_map-i qəsdən pis hala salan testlər ("anti-hash hack") hazırlayırlar: standart hash funksiyası məlumdur, hamısı bir yuvaya düşən giriş qurmaq olur, hansı ki, kodları TLE almağa məruz qoyur. Bunun qarşısını almaq üçün ya map işlədin, ya da öz random hash-inizi yazın:

struct custom_hash {
    size_t operator()(uint64_t x) const {
        x += 0x9e3779b97f4a7c15 + (uint64_t)chrono::steady_clock::now().time_since_epoch().count();
        x = (x ^ (x >> 30)) * 0xbf58476d1ce4e5b9;
        x = (x ^ (x >> 27)) * 0x94d049bb133111eb;
        return x ^ (x >> 31);
    }
};
unordered_map<long long, int, custom_hash> yer;

Random komponent sayəsində hazır anti-test işləmir. CSES-də isə belə bir problem yoxdur, adi unordered işlətmək çox zaman problem yaratmır.

String hashing: string-i ədədə çevirmək​

Hashing-in ikinci böyük tətbiqi isə stringləri müqayisə etmək üçündür. İki uzun string-i hərf-hərf müqayisə O(uzunluq) olur, lakin hash-ləri müqayisə etmək O(1) olur.

Polinomial hash: sətri p əsaslı ədəd kimi oxu (mod M):

hash(s) = (s[0] * p⁰ + s[1] * p¹ + ... + s[k] * pᵏ) mod M
const long long P = 31, M = 1e9 + 9;

long long strHash(const string &s) {
    long long h = 0, pq = 1;
    for (char c : s) {
        h = (h + (c - 'a' + 1) * pq) % M;
        pq = pq * P % M;
    }
    return h;
}

Fərqli string-lər eyni hash ala bilərmi? Bəli, və bu toqquşmanın ehtimalı ~1/M-dir. M ≈ 10⁹ və müqayisələr çox olanda risk hiss olunur, buna görə də çətin məsələlərdə iki fərqli (P, M) cütü ilə qoşa hash istifadə olunur.

Nümunə: neçə fərqli substring? (Good Substrings)​

Bütün substringləri unordered_set-ə atmaq üçün onları saxlamaq yox, hash-lərini saxlamaq bəsdir:

unordered_set<long long> s;
for (int i = 0; i < n; i++) {
    long long h = 0, pq = 1;
    for (int j = i; j < n; j++) {
        h = (h + (s[j] - 'a' + 1) * pq) % M;
        pq = pq * P % M;
        s.insert(h);
    }
}
cout << s.size();

O(n²) substring + hər biri O(1) yenilənən hash = cəmi O(n²). String-ləri saxlasaydıq O(n³) yaddaş/vaxt olardı.

Çalışmalar​

Çalışma 1. İki massivin ortaq elementlərinin sayını (təkrarlar olmadan) unordered_set-lə O(n + m) zaman səmərəliyi ilə tapın.

Həlli
unordered_set<int> A(a.begin(), a.end());
unordered_set<int> s;
for (int x : b)
    if (A.count(x)) s.insert(x);
cout << s.size();

Çalışma 2. Stringlərin siyahısında anaqram qruplarının sayını tapın. Anaqramlar eyni hərflərin yerdəyişməsi ilə yaranan fərqli sözlərə deyilir. Məsələn, "listen" və "silent" anaqramlara nümunə ola bilər.

Həlli
unordered_map<string, int> mp;
for (string s : words) {
    sort(s.begin(), s.end());
    mp[s]++;
}
cout << mp.size();

Çalışma 3. "s" sətrində uzunluğu k olan neçə fərqli substring var? Polinomial hash-lə O(n) həll qurun (rolling hash). İpucu: "sliding windows" işlədin.

Həlli
// hash(s[i+1..i+k]) = (hash(s[i..i+k-1]) - s[i]) / p
// prefiks hashləri saxlayıb çıxmaq daha rahatdır:
// pre[i] = hash(s[0..i-1]), h(l,r) = (pre[r+1] - pre[l] + M) % M * pinv[l] % M

unordered_set<long long> s;
long long h = 0, pk = 1;
for (int i = 0; i < k; i++) { h = (h * P + s[i]) % M; pk = pk * P % M; }
s.insert(h);
for (int i = k; i < n; i++) {
    h = (h * P + s[i] - pk * s[i - k] % M + M * P) % M;
    s.insert(h);
}
cout << s.size();
note

Burada hash "böyük bit soldadır" formasında yazılıb (h * P + yeni hərf)/

Tez-tez edilən səhvlər​

  1. Codeforces-da adi unordered_map-lə hack yemək;
  2. unordered_set-də lower_bound axtarmaq;
  3. String hash-də mod çıxmasında mənfi qalıq (+M-i unutmaq);
  4. Hash bərabərliyini həmişə işlədiyini fikirləşmək;
  5. unordered_map-in []-i ilə yoxlayıb açar yaratmaq.
