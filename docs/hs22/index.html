Həll saatı 21. Rekursiya və backtracking

Rekursiya — funksiyanın öz-özünü çağırmasıdır. İlk baxışda qəribə görünür, amma arxasında sadə fəlsəfə var: böyük məsələni eyni formalı kiçik məsələyə çevir, ən kiçik halı əl ilə həll et. Rekursiyanı dərindən başa düşmək — DP, qraflar (DFS) və ağac alqoritmlərinin hamısına açar verir.

İlk rekursiv funksiya​

#include <iostream>
using namespace std;

long long faktorial(int n) {
    if (n <= 1) return 1;
    return n * faktorial(n - 1);
}

int main() {
    cout << faktorial(5);
    return 0;
}

Hər rekursiv funksiyada iki hissə mütləqdir:

  1. Base case — rekursiyasız cavab verilən ən kiçik hal (n <= 1)
  2. Kiçilmə addımı — özünü daha kiçik arqumentlə çağırmaq

faktorial(5) çağırışını izləyək:

faktorial(5) → 5 * faktorial(4)
             → 5 * (4 * faktorial(3))
             → 5 * (4 * (3 * faktorial(2)))
             → 5 * (4 * (3 * (2 * faktorial(1))))
             → 5 * (4 * (3 * (2 * 1))) = 120

Hər çağırış stack-də (7-ci dərsdəki funksiya çağırış steki) gözləyir, base case-ə çatanda cavablar geri "sarılır".

Base case unutsanız

Funksiya özünü sonsuz çağırır, stack dolur və proqram çökür: stack overflow. Rekursiya yazanda ilk sual: "base case nədir və ona mütləq çatırammı?"

Rekursiyanın həndəsəsi: çağırış ağacı​

Fibonacci ilə fərqli mənzərə görünür:

long long fib(int n) {
    if (n <= 1) return n;
    return fib(n - 1) + fib(n - 2);
}

fib(5) çağırış ağacı:

                fib(5)
              /        \
          fib(4)        fib(3)
         /      \      /      \
     fib(3)   fib(2) fib(2)  fib(1)
     /    \    ...    ...
  fib(2) fib(1)

fib(3) iki dəfə, fib(2) üç dəfə hesablanır! Çağırışların sayı eksponensial böyüyür: fib(50) artıq günlərlə işləyər. Bu israfçılığın dərmanı — memoization — növbəti dərsin mövzusudur. Bu dərsdə isə eksponensiallığın qanuni istifadəsinə baxırıq: bütün variantları gəzmək.

Bütün variantları rekursiya ilə gəzmək​

20-ci dərsdə alt çoxluqları bitmask ilə gəzdik. Eyni işi rekursiya daha təbii edir: hər element üçün iki qərar — götür / götürmə:

int n;
vector<int> a;
vector<int> secilen;

void go(int i) {
    if (i == n) {
        for (int x : secilen) cout << x << ' ';
        cout << '\n';
        return;
    }
    // Qerar 1: a[i]-ni goturme
    go(i + 1);

    // Qerar 2: a[i]-ni gotur
    secilen.push_back(a[i]);
    go(i + 1);
    secilen.pop_back();
}

push_back → rekursiya → pop_back cütlüyünə diqqət: qərarı verdik, dərinə getdik, qayıdanda ləğv etdik. Bu "izini təmizləmə" hərəkətinə görə üsul backtracking adlanır.

Backtracking: ağıllı tam axtarış​

Backtracking = variantları rekursiv gəzmək + perspektivsiz budaqları erkən kəsmək. Klassik nümunə — n vəzir məsələsi (CSES 1624: 8 * 8 taxtada 8 vəzir, bəzi xanalar qadağandır):

#include <bits/stdc++.h>
using namespace std;

int n = 8;
vector<string> taxta(8);
bool sutun[8], diag1[15], diag2[15];   // hansi xetler tutulub?
long long say = 0;

void qoy(int setir) {
    if (setir == n) { say++; return; }          // 8 vezir qoyuldu
    for (int s = 0; s < n; s++) {
        if (taxta[setir][s] == '*') continue;    // qadagan xana
        if (sutun[s] or diag1[setir + s] or diag2[setir - s + 7]) continue;

        sutun[s] = diag1[setir + s] = diag2[setir - s + 7] = true;
        qoy(setir + 1);
        sutun[s] = diag1[setir + s] = diag2[setir - s + 7] = false;  // backtrack
    }
}

int main() {
    for (auto &row : taxta) cin >> row;
    qoy(0);
    cout << say;
    return 0;
}

Üç açar fikir:

  1. Sətir-sətir qoyuruq — hər sətirdə tam bir vəzir olmalıdır, deməli qərar = "bu sətirdə hansı sütun?"
  2. Toqquşmanı O(1) yoxlayırıq: eyni sütun s, eyni "/" diaqonalı setir+s, eyni "" diaqonalı setir-s+7.
  3. Erkən kəsmə: toqquşan sütuna heç girmirik — 8⁸ ≈ 1.7 * 10⁷ variant əvəzinə minlərlə qalır.
Backtracking template-i
void go(condition):
    if (tam hell) → qeyd et, return
    for (hər mümkün addım):
        if (addım perspektivsizdir) continue      // kəsmə
        addımı at
        go(yeni condition)
        addımı geri al                            // backtrack

Bu skeleti əzbər bilin, vəzirlər, labirintlər, sudoku, permutasiyalar və s. bu qaydaya uyğun həll olunur.

Permutasiyalar rekursiya ilə​

20-ci dərsdə next_permutation gördük; rekursiv qurmaq da vacib bacarıqdır (CSES 1070):

int n;
vector<int> yol;
bool istifade[9];

void qur() {
    if ((int)yol.size() == n) {
        for (int x : yol) cout << x << ' ';
        cout << '\n';
        return;
    }
    for (int x = 1; x <= n; x++) {
        if (istifade[x]) continue;
        istifade[x] = true;
        yol.push_back(x);
        qur();
        yol.pop_back();
        istifade[x] = false;
    }
}

Leksikoqrafik sıra pulsuz alınır, çünki x-i 1-dən n-ə sıra ilə sınayırıq.

Hanoi qülləsi​

Rekursiv düşüncənin şedevri (CSES 2165): n diski A-dan C-yə keçir, böyük disk kiçiyin üstünə qoyula bilməz.

Rekursiv fikir: n diski A→C keçirmək = (n-1 diski A→B) + (ən böyük diski A→C) + (n-1 diski B→C):

void hanoi(int n, int A, int C, int B) {
    if (n == 0) return;
    hanoi(n - 1, A, B, C);
    cout << A << ' ' << C << '\n';
    hanoi(n - 1, B, C, A);
}

Gedişlərin sayı 2ⁿ − 1 — və bundan azı mümkün deyil. Üç sətirlik kod, tam düzgün həll: rekursiyanın gücü budur.

Çalışmalar​

Çalışma 1. Rekursiv guc(a, n) = aⁿ yazın (n ≥ 0). Sonra sürətli versiya: n cütdürsə guc(a * a, n/2).

Həlli
long long guc(long long a, int n) {
    if (n == 0) return 1;
    if (n % 2 == 0) return guc(a * a, n / 2);
    return a * guc(a, n - 1);
}

Sürətli versiya O(log n) — bu, "binary exponentiation", 27-ci dərsdə mod arifmetikası ilə birləşəcək.

Çalışma 2. Sətrin bütün alt ardıcıllıqlarını (subsequence) çap edin. "abc" → a, b, c, ab, ac, bc, abc (boş olmayan).

Həlli
string s, yol;

void go(int i) {
    if (i == (int)s.size()) {
        if (!yol.empty()) cout << yol << '\n';
        return;
    }
    go(i + 1);               // s[i]-ni atla
    yol.push_back(s[i]);     // s[i]-ni gotur
    go(i + 1);
    yol.pop_back();
}

Çalışma 3. 1-dən n-ə qədər ədədlərdən cəmi t olan neçə alt çoxluq var? (n ≤ 20). Rekursiya + kəsmə: cəm t-ni aşanda dayanın.

Həlli
int n, t;
long long say = 0;

void go(int i, int cem) {
    if (cem == t) { say++; }         // tapdıq (davam etmək olar, amma cem artacaq)
    if (i > n or cem >= t) return;   // kesme: ele bil qalani goturmek menasizdir
    go(i + 1, cem);                 // i-ni goturme
    go(i + 1, cem + i);             // i-ni gotur
}

Kəsmə (cem >= t) müsbət ədədlərdə işləyir: cəm artıq t-yə çatıbsa, əlavə götürmək yalnız artırar.

Tez-tez edilən səhvlər​

  1. Base case unutmaq və ya səhv qoymaq → stack overflow.
  2. Backtrack addımını (pop_back / flag-i geri almaq) unutmaq — ən çox edilən səhv!
  3. Qlobal vəziyyəti dəyişib geri qaytarmamaq.
  4. Rekursiyanın dərinliyini hesablamamaq: 10⁶ dərinlik stack-i böyüdür (adətən limit ~10⁵-10⁶ çağırış dərinliyidir; dərin xətti rekursiyanı dövrə çevirin).
  5. fib üslubunda təkrar hesablamaların eksponensial böyüdüyünü görməmək — bunun həlli növbəti dərsdədir.
