# Həll saatı 9. Sadə riyaziyyat

CP-də "riyaziyyat" çox vaxt ali riyaziyyat deyil, sadəcə tam bölmə, qalıq, yuvarlaqlaşdırma və overflow kimi anlayışlardır. Bu dərsdə həmin anlayışlara nəzər salacayıq.

Bu dərsdə öyrənəcəyik:
- tam bölmə və qalıqlar
- yuxarı həddə yuvarlaqlaşdırma
- overflow

## Tam bölmə və qalıq

```cpp
7 / 2    // 3   (tam bölmə)
7 % 2    // 1   (qalıq)
-7 / 2   // -3  (sıfıra doğru yuvarlaqlaşdır)
-7 % 2   // -1  (qalıq mənfi ola bilər)
```

:::danger[Mənfi qalıq]
C++-da mənfi ədədin qalığı mənfi ola bilər. Həmişə 0 . . . m−1 aralığında qalıq lazımdırsa: `((x % m) + m) % m` kimi yazmaq olar. Dövri indekslərdə (məsələn saat, dövri massiv) bu düstur mütləqdir.
:::

## Yuxarı yuvarlaqlaşdırma

"n nəfəri k yerlik masalara oturtmaq üçün neçə masa lazımdır?" ⌈n/k⌉ dənə. `double` tipinə keçmədən:

```cpp
long long cv = (n + k - 1) / k;
```

Niyə işləyir: n k-ya bölünürsə, `k - 1` əlavəsi nəticəni dəyişmir. Bölünmürsə, bir vahid yuxarı itələyir. 

Codeforce-da Theatre Square məsələsi: `n * m` meydanı `a * a` daşlarla örtmək:

```cpp
#include <iostream>
using namespace std;

int main() {
    long long n, m, a;
    cin >> n >> m >> a;
    cout << ((n + a - 1) / a) * ((m + a - 1) / a);
    return 0;
}
```

n, m ≤ 10⁹ ➝ cavab 10¹⁸-ə qədər. Mütləq `long long` istifadə olunmalıdır. 

## Daşma (overflow)

Tiplərin sərhədləri:

| Tip | Diapazon | Təqribi |
|---|---|---|
| `int` | ±2.1 * 10⁹ | "2 milyard" |
| `long long` | ±9.2 * 10¹⁸ | "9 kvintilyon" |

Önəmli məqam: hesablama aralıq nəticədə overflow edir:

```cpp
int n = 100000;
long long x = n * n;              // Səhv: n * n int-də hesablanır və overflow edir
long long x = (long long)n * n;   // doğru 
long long x = 1LL * n * n;        // qısa yol
```

:::danger[Əməliyyatın tipi]
Toplama, çıxma, vurma, və s. əməllərdə əməliyyat birinci ədədin tipinə görə aparılır. Yəni, birinci ədəd `int` olsa overflow riski ilə qarşı qarşıya qalırsınız.
:::

## Faiz və hissə

Kəsr hesablamada mümkün qədər axırda bölün:

```cpp
// a ədədinin b-yə nisbətinin faizlə ifadəsi
double p = 100.0 * a / b;    // 100.0: double tipi, tam bölmədən qaçır
```

Kəsrləri müqayisə edərkən bölmədən uzaq durmaq daha yaxşıdır: `a/b < c/d` yerinə `a*d < c*b` (diqqət işarələrə). `double` dəqiqliyi məhduddur, ona görə də bərabərsizliyi yoxlamaq üçün məsləhətli deyil.

## Arifmetik silsilə

1-dən n-ə qədər cəm: `n * (n + 1) / 2`. Diqqət: vurma overflow edə bilər:

```cpp
long long sum = (long long)n * (n + 1) / 2;
```

l-dən r-ə cəm: `sum(r) - sum(l - 1)`.

## Rəqəmlərlə iş 

Əvvəlki dərsdən bilirik ki, `n % 10` ədədin təklik mərtəbəsi, `n /= 10` ədədin 10-a bölündüyünü göstərir. Eyni nəticəni ədədi stringə çevirməklə də ala bilərik:

```cpp
string s = to_string(n);    // ədəd ➝ string
int m = stoi(s);            // string ➝ int
long long k = stoll(s);     // string ➝ long long
```

"Rəqəmlərinin sırası", "palindrom yoxlamaq", "müəyyən rəqəmin sayı" tipli məsələlərdə string yolu çox vaxt daha qısadır.

## Həndəsi düsturlar (minimal dəst)

- Düzbucaqlının: sahəsi `a*b`, perimetri `2*(a+b)`
- İki nöqtə arası məsafə: `sqrt((x1-x2)² + (y1-y2)²)`, amma müqayisə üçün kvadratları olduğu kimi saxlayın, `sqrt` işlətməyin:

```cpp
long long d2 = (long long)(x1-x2)*(x1-x2) + (long long)(y1-y2)*(y1-y2);
// d2 < r*r  ⟺  məsafə < r   (heç bir double xətası yoxdur!)
```

Dəqiqlik problemlərini yaşamamaq üçün tam ədədlər işlətmək məqsədəuyğundur.  
## Çalışmalar

**Çalışma 1.** n saniyəni "saat:dəqiqə:saniyə" formatına çevirin.

<details>
<summary>Həlli</summary>

```cpp
long long n;
cin >> n;
cout << n / 3600 << ":" << (n % 3600) / 60 << ":" << n % 60;
```

</details>

**Çalışma 2.** a və b verilir. a-dan böyük və ya bərabər olan ən kiçik b-yə bölünən ədədi tapın.

<details>
<summary>Həlli</summary>

```cpp
long long cv = (a + b - 1) / b * b;
```

Yuxarı yuvarlaqlaşdırılmış bölmə * b. Məsələn a=13, b=5: (13+4)/5 = 3, 3 * 5 = 15.

</details>

**Çalışma 3.** n! ədədinin sonunda neçə sıfır var? (Qeyd: ədəd çox böyük olduqda faktorialını hesablamaq mümkün deyil)

<details>
<summary>Həlli</summary>

```cpp
long long cv = 0;
for (long long p = 5; p <= n; p *= 5)
    cv += n / p;
cout << cv;
```

Sondakı sıfırlar 10 = 2 * 5 vuruqlarından yaranır. 5-lər 2-lərdən azdır, ona görə də 5-ləri sayırıq: n/5 ədəd 5-ə bölünən, n/25 əlavə 5 verən, və s.

</details>

## Tez-tez edilən səhvlər

1. `n * n` int daşması(overflow);
2. Mənfi ədədlərin qalığını 0 . . . m−1 sanmaq;
3. Yuxarı yuvarlaqlaşdırma əvəzinə adi bölmə;
4. `double` ilə bərabərlik yoxlamaq;
5. `sqrt` istifadə edib dəqiqliyi itirmək.
