# Həll saatı 24. Two pointers

O(n²) dövr cütü çox vaxt belə görünür: "hər sol uc üçün uyğun sağ ucu tap". **Two Pointers** üsulu müşahidə edir ki, çox məsələdə sol uc irəlilədikcə sağ uc **geri qayıtmır** — onda hər iki pointer massivi cəmi bir dəfə gəzir: O(n²) → O(n).

## Klassika: sıralanmış massivdə cüt cəmi

a sıralanıb; `a[i] + a[j] = x` cütü axtarırıq. Göstəricilər iki ucdan:

```cpp
int sol = 0, sag = n - 1;
while (sol < sag) {
    long long cem = a[sol] + a[sag];
    if (cem == x) { /* tapdıq: sol, sag */ break; }
    if (cem < x) sol++;      // cem kicikdir → boyut
    else sag--;              // cem böyükdur → kicilt
}
```

Niyə düzgündür? `cem < x` olduqda `a[sol]` heç bir cütdə işə yaramır: ən böyük mümkün tərəf-müqabili ilə belə cəm kiçikdir. Deməli onu birdəfəlik atmaq təhlükəsizdir. Hər addım bir göstəricini tərpədir → maksimum 2n addım.

CSES 1640 (Sum of Two Values) bunun üstündə qurulur — yalnız **ilkin indeksləri** qorumaq lazımdır, çünki sıralama yerləri dağıdır:

```cpp
vector<pair<long long,int>> a(n);          // {qiymet, ilkin indeks}
for (int i = 0; i < n; i++) {
    cin >> a[i].first;
    a[i].second = i + 1;
}
sort(a.begin(), a.end());

int sol = 0, sag = n - 1;
while (sol < sag) {
    long long cem = a[sol].first + a[sag].first;
    if (cem == x) {
        cout << a[sol].second << ' ' << a[sag].second;
        return 0;
    }
    cem < x ? sol++ : sag--;
}
cout << "IMPOSSIBLE";
```

Bu məsələni artıq üç yolla bilirsiniz: map (17), sırala+binar axtarış (19), two pointers (24). Üçü də O(n log n) civarı — amma two pointers sıralamadan sonra O(n) və ən az kod.

## Sliding windows:

İteratorlar eyni ucdan başlayır, aralarındakı aralıq [sol, sag] böyüyüb-kiçilir. Şərt: Aralığın "yaxşılığı" monoton olmalıdır: böyütdükcə eyni cür işləyirsə, üsul işləyir.

### Nümunə: cəmi ≤ S olan ən uzun aralıq (müsbət ədədlər)

```cpp
long long cem = 0;
int sol = 0, cavab = 0;
for (int sag = 0; sag < n; sag++) {
    cem += a[sag];
    while (cem > S) {
        cem -= a[sol];
        sol++;
    }
    cavab = max(cavab, sag - sol + 1);
}
```

Skelet həmişə eynidir:

```
for (sag = 0..n-1):
    sag elementi pencereye elave et
    while (pencere pisdir): sol elementi cixar, sol++
    cavabi yenile
```

`sol` heç vaxt geri getmir → hər element pəncərəyə bir dəfə girib bir dəfə çıxır → O(n).

:::danger[Monotonluq şərtdir]
Mənfi ədədlər varsa "cəm > S → kiçilt" məntiqi sınır: pəncərəni kiçiltmək cəmi **artıra** bilər. Belə halda two pointers yox, prefix + map (23-cü dərs) işlədin. Üsulu tətbiq etməzdən əvvəl həmişə soruşun: *aralıq böyüdükcə şərt birtərəfli dəyişirmi?*
:::

### Nümunə: Subarray Sums I (CSES 1660)

Cəmi **düz x** olan subarray sayı (ədədlər müsbət):

```cpp
long long cem = 0, cavab = 0;
int sol = 0;
for (int sag = 0; sag < n; sag++) {
    cem += a[sag];
    while (cem > x) cem -= a[sol++];
    if (cem == x) cavab++;
}
cout << cavab;
```

Müsbətlik sayəsində hər sağ uc üçün cəmi ≤ x edən minimal aralıq unikaldır — `while`-dan çıxanda ya düz x-dir, ya kiçikdir.

### Nümunə: Playlist (CSES 1141)

Təkrarsız ən uzun ardıcıl parça:

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    int n;
    cin >> n;

    vector<int> a(n);
    for (auto &x : a) cin >> x;

    set<int> pencere;
    int sol = 0, cavab = 0;
    for (int sag = 0; sag < n; sag++) {
        while (pencere.count(a[sag])) {     // tekrar → soldan tekrari at
            pencere.erase(a[sol]);
            sol++;
        }
        pencere.insert(a[sag]);
        cavab = max(cavab, sag - sol + 1);
    }
    cout << cavab;
    return 0;
}
```

## İki sıralanmış massivin birləşməsi (merge)


```cpp
int i = 0, j = 0;
while (i < n or j < m) {
    if (j == m or (i < n and a[i] <= b[j])) cout << a[i++] << ' ';
    else cout << b[j++] << ' ';
}
```

"İki siyahını sinxron gəzmək" — ortaq elementlər, kəsişmə, birləşmə məsələlərinin hamısında bu skelet.

## Çalışmalar

**Çalışma 1.** Sıralanmış massivdə `a[j] - a[i] = x` (i < j) cütü tapın.

<details>
<summary>Həlli</summary>

```cpp
int i = 0;
bool var = false;
for (int j = 0; j < n; j++) {
    while (a[j] - a[i] > x) i++;        // ferq böyükdur → i-ni yaxinlashdir
    if (i < j and a[j] - a[i] == x) var = true;
}
```

Fərq məsələsində ucdan-uca yox, eyni istiqamətli göstəricilər işləyir: j böyüdükcə uyğun i də yalnız böyüyür.

</details>

**Çalışma 2.** Hər biri ≤ k olan elementlərdən ibarət ən uzun ardıcıl parçanı tapın. (Pəncərəsiz, tək gedişlə də olur)

<details>
<summary>Həlli</summary>

```cpp
int cari = 0, cavab = 0;
for (int i = 0; i < n; i++) {
    if (a[i] <= k) cari++;
    else cari = 0;
    cavab = max(cavab, cari);
}
```

Şərt hər elementə ayrıca baxılırsa, aralıq yerinə sadə sayğac bəsdir. Two pointers-i yalnız pəncərənin **cəmi/tərkibi** vacib olanda işlədin — alət seçimi də bacarıqdır.

</details>

**Çalışma 3.** Books (CF 279B): i-ci kitab t[i] dəqiqədir; ardıcıl kitablar oxumaqla T dəqiqəyə maksimum neçə kitab? 

<details>
<summary>Həlli</summary>

```cpp
long long cem = 0;
int sol = 0, cavab = 0;
for (int sag = 0; sag < n; sag++) {
    cem += t[sag];
    while (cem > T) cem -= t[sol++];
    cavab = max(cavab, sag - sol + 1);
}
```

</details>

## Tez-tez edilən səhvlər

1. Sıralanmamış massivdə uc-uca göstəricilər işlətmək.
2. Mənfi ədədlərdə sliding windows tətbiq etmək (monotonluq pozulur).
3. `sol`-u pəncərədən çıxarmadan irəli çəkmək (cəmi/seti yeniləməyi unutmaq).
4. `sag - sol + 1` uzunluğunda +1-i unutmaq.
5. Sum of Two Values-də ilkin indeksləri sıralamadan əvvəl saxlamamaq.
