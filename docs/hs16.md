# Həll saatı 15. Stack, queue və deque

Bu üç struktur — elementlərin **hansı ucdan girib-çıxdığını** məhdudlaşdıran siyahılardır. Məhdudiyyət zəiflik deyil, gücdür: düzgün struktur seçilən məsələ öz-özünə həll olunur.

## Stack — LIFO

**Son giren birinci çıxır** (Last In, First Out) — boşqab qülləsi kimi.

```cpp
#include <stack>

stack<int> st;
st.push(3);      // qoy
st.push(7);
cout << st.top();   // 7 — zirvedeki (SILMIR)
st.pop();           // zirvedekini sil (QAYTARMIR!)
st.size(); st.empty();
```

Diqqət: `pop()` qiymət **qaytarmır** — əvvəl `top()`, sonra `pop()`.

Stack harada təbii yaranır? "Sonuncunu geri al" (undo), funksiya çağırışları, **mötərizə balansı**:

### Klassik: mötərizə yoxlaması

`([]{})` düzgündür, `([)]` səhvdir:

```cpp
bool duzgundur(string &s) {
    stack<char> st;
    for (char c : s) {
        if (c == '(' or c == '[' or c == '{') {
            st.push(c);
        } else {
            if (st.empty()) return false;         // baglanan var, acan yox
            char acan = st.top(); st.pop();
            if ((c == ')' and acan != '(') or
                (c == ']' and acan != '[') or
                (c == '}' and acan != '{')) return false;
        }
    }
    return st.empty();    // acilib baglanmayan qalmasin!
}
```

İdeya: hər açılan mötərizə "gözləyir"; bağlanan gələndə **ən son açılanla** birləşməlidir — bu "ən son" sözü elə stack-in özüdür. İki edge case-ə baxın: boş stack-dən pop, sonda boş olmayan stack.

## Queue — FIFO

**Birinci girən birinci çıxır** (First In, First Out) — növbə.

```cpp
#include <queue>

queue<int> q;
q.push(3);          // sona qoy
q.push(7);
cout << q.front();  // 3 — evvvldeki
q.pop();            // evveldekini sil
q.back();           // sondaki
```

Queue harada? Emal növbələri, "gözləyənlər siyahısı" simulyasiyaları və — ən önəmlisi — **BFS** (labirintdə ən qısa yol). Qraf dərslərinə çatanda queue çörək kimi lazım olacaq; indi mexanikasına alışın.

### Nümunə: sənəd növbəsi

Printerə n sənəd gəlir; hər addımda ya yeni sənəd gəlir (`1 x`), ya növbədəki çap olunur (`2`):

```cpp
queue<int> q;
int emr;
while (cin >> emr) {
    if (emr == 1) {
        int x; cin >> x;
        q.push(x);
    } else {
        if (q.empty()) cout << "bosh\n";
        else { cout << q.front() << '\n'; q.pop(); }
    }
}
```

## Deque — iki başlı növbə

Hər iki ucdan O(1) əlavə/silmə:

```cpp
#include <deque>

deque<int> d;
d.push_back(1);   d.push_front(2);   // [2, 1]
d.back();         d.front();
d.pop_back();     d.pop_front();
d[i];             // indeksleme de var!
```

Deque = stack + queue + indeksləmə bir yerdə. Bəs niyə həmişə deque işlətmirik? Vector-dan bir az yavaşdır və yaddaşı dağınıqdır — yalnız iki uc lazımdırsa seçin.

Klassik tətbiq: **sliding windows minimum**, hansı ki gələn dərslərdə öyrənəcəyik:

### Nümunə: kart düzülüşü

1-dən n-ə kartları növbə ilə gah sola, gah sağa qoyuruq:

```cpp
deque<int> d;
for (int i = 1; i <= n; i++) {
    if (i % 2 == 1) d.push_front(i);
    else d.push_back(i);
}
for (int x : d) cout << x << ' ';
// n=5: 5 3 1 2 4
```

## Hansını seçməli?

| Lazımdır | Struktur |
|---|---|
| yalnız son element | `stack` |
| yalnız ilk element (gəlmə sırası) | `queue` |
| hər iki uc | `deque` |
| ixtiyari indeks + sondan əməliyyat | `vector` |

Məsələni oxuyanda özünüzə sual verin: **elementlər hansı sırada emal olunur?** "Ən son gələn əvvəl" eşidirsinizsə — stack; "gəldiyi sırada" — queue.

## Çalışmalar

**Çalışma 1.** Sətirdəki mətndə "geri al" əməliyyatını simulyasiya edin: hər `#` simvolu özündən əvvəlki hərfi silir. `ab#c##d` → `d`... yox, əl ilə yoxlayın!

<details>
<summary>Həlli</summary>

```cpp
string s, netice;
cin >> s;
for (char c : s) {
    if (c == '#') { if (!netice.empty()) netice.pop_back(); }
    else netice += c;
}
cout << netice;
```

`ab#c##d`: a, ab, a, ac, a, "", d → cavab `d`. Burada `string`-in özü stack rolundadır (`+=` = push, `pop_back` = pop) — struktur anlayışdır, konkret tip deyil.

</details>

**Çalışma 2.** Queue-da n ədəd var. Yalnız queue əməliyyatları ilə (və bir köməkçi queue ilə) minimumu **önə** gətirin, qalanların sırası saxlanmaqla.

<details>
<summary>Həlli</summary>

```cpp
// 1-ci gediş: minimumu tap (queue-nu firladaraq)
int n = q.size(), mn = 2000000000;
for (int i = 0; i < n; i++) {
    mn = min(mn, q.front());
    q.push(q.front()); q.pop();
}
// 2-ci gediş: minimumdan başqa hamısı köməkçi queue-ya
queue<int> k;
bool goturuldu = false;
for (int i = 0; i < n; i++) {
    if (q.front() == mn and !goturuldu) goturuldu = true;
    else k.push(q.front());
    q.pop();
}
q.push(mn);
while (!k.empty()) { q.push(k.front()); k.pop(); }
```

</details>

**Çalışma 3.** Sətir verilir. Deque ilə onun palindrom olub-olmadığını yoxlayın.

<details>
<summary>Həlli</summary>

```cpp
deque<char> d;
for (char c : s) d.push_back(c);
bool ok = true;
while (d.size() >= 2) {
    if (d.front() != d.back()) ok = false;
    d.pop_front(); d.pop_back();
}
```

</details>

## Tez-tez edilən səhvlər

1. Boş strukturda `top()/front()/pop()`-un verdiyi runtime error;
2. `pop()`-un qiymət qaytardığını sanmaq;
3. Mötərizə məsələsində sondakı `st.empty()` yoxlamasını unutmaq;
4. Stack/queue-nu gəzmək üçün boşaltmaq və verilənləri itirmək;
5. Deque-də indeksləmə var deyə onu vector əvəzinə hər yerdə işlətmək.
