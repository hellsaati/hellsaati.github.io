# Həll saatı 19. Binary search

Lüğətdə söz axtaranda hər səhifəni bir-bir çevirmirsiniz — ortadan açıb "əvvəldədir, ya sonra?" deyə yarıya bölürsünüz. **Binar axtarış** məhz budur: hər addımda axtarış sahəsini yarıya endirmək. 10⁹ elementli sahə — cəmi ~30 addım.

İki üzü var:

1. Sıralanmış massivdə qiymət axtarmaq
2. **Cavab üzərində binar axtarış** — olimpiadaların ən güclü üsullarından biri

## Sıralanmış massivdə axtarış

```cpp
// a sıralanıb (artan). x-i axtarırıq.
int sol = 0, sag = n - 1, cavab = -1;
while (sol <= sag) {
    int orta = sol + (sag - sol) / 2;
    if (a[orta] == x) { cavab = orta; break; }
    if (a[orta] < x) sol = orta + 1;
    else sag = orta - 1;
}
```

Hər addımda `[sol, sag]` intervalı yarıya düşür → O(log n).

:::danger[Şərt: massiv sıralanmış olmalıdır!]
Sıralanmamış massivdə binar axtarış **səhv cavab** verir (xəta yox — sadəcə yanlış nəticə). Bu, ən sinsi bug növüdür.
:::

`orta = sol + (sag - sol) / 2` yazılışı `(sol + sag) / 2`-dən üstündür: cəm daşa bilməz.

## STL: lower_bound / upper_bound

Praktikada əl ilə yazmaq əvəzinə STL işlədirik:

```cpp
vector<int> a = {2, 4, 4, 7, 10};   // sıralanıb!

// lower_bound: >= x olan ILK elementin iteratoru
int i = lower_bound(a.begin(), a.end(), 4) - a.begin();   // 1

// upper_bound: > x olan ILK element
int j = upper_bound(a.begin(), a.end(), 4) - a.begin();   // 3

// x-in sayi:
int say = j - i;    // 2 dene 4 var

// x varmi?
bool var = binary_search(a.begin(), a.end(), 7);   // true
```

`- a.begin()` iteratoru indeksə çevirir. `[lower_bound, upper_bound)` intervalı — x-ə bərabər elementlərin dəqiq sahəsi. Bu üçlüyü əzbər bilin.

### Nümunə: Interesting Drink (CF 706B)

n mağazada qiymətlər verilir; hər sorğu üçün "m manatla neçə mağazadan almaq olar?":

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    int n;
    cin >> n;

    vector<int> a(n);
    for (auto &x : a) cin >> x;
    sort(a.begin(), a.end());

    int q;
    cin >> q;
    while (q--) {
        int m;
        cin >> m;
        cout << upper_bound(a.begin(), a.end(), m) - a.begin() << '\n';
    }
    return 0;
}
```

Sırala bir dəfə O(n log n), hər sorğu O(log n). Sorğusuz-sıralamasız yol O(n * q) olardı — 10¹⁰ əməliyyat, keçməz.

## Cavab üzərində binar axtarış

**"Cavab t ola bilərmi?"** sualına asan cavab verə biliriksə və bu sual **monotondursa** (t işləyirsə, t+1 də işləyir), cavabın özünü binar axtarışla tapırıq.

### Nümunə: Factory Machines (CSES 1620)

n maşın var, i-cisi bir məhsulu k[i] saniyəyə düzəldir. t məhsulu ən azı nə qədər vaxta düzəltmək olar?

Sualı çevirək: **"x saniyədə t məhsul düzələrmi?"** — asan: hər maşın x saniyədə `x / k[i]` məhsul düzəldir, cəmləyib t ilə müqayisə edirik. Və monoton: x saniyədə düzəlirsə, x+1-də də düzələr.

```cpp
#include <bits/stdc++.h>
using namespace std;

int n;
long long t;
vector<long long> k;

bool olur(long long x) {
    long long cem = 0;
    for (long long ki : k) {
        cem += x / ki;
        if (cem >= t) return true;
    }
    return cem >= t;
}

int main() {
    cin >> n >> t;
    k.resize(n);
    for (auto &x : k) cin >> x;

    long long sol = 1, sag = 2e18;
    while (sol < sag) {
        long long orta = sol + (sag - sol) / 2;
        if (olur(orta)) sag = orta;
        else sol = orta + 1;
    }
    cout << sol;
    return 0;
}
```

Bu template-i yaxşı öyrənin:

```
sol = "mutleq olmur"-dan böyük ilk mumkun qiymet
sag = "mutleq olur" qiymet
while (sol < sag): olursa sag = orta, olmursa sol = orta + 1
cavab = sol
```

:::tip[Binar axtarışı nə vaxt düşünməli?]
Məsələdə "minimum maksimumu", "maksimum minimumu", "ən azı nə qədər", "ən çoxu nə qədər" sözləri keçirsə və yoxlama funksiyası qurmaq asandırsa — 90% cavab üzərində binar axtarışdır.
:::

## while (sol `<` sag) vs while (sol `<=` sag)

İki düzgün üslub var; qarışdırmayın:

- **Qiymət axtarışı** (tapıldı/tapılmadı): `sol <= sag`, tapılanda `break`.
- **Ən kiçik "olur" qiyməti**: `sol < sag`, `sag = orta` / `sol = orta + 1`. Dövr bitəndə `sol == sag` — cavab.

Sonsuz dövrə düşürsünüzsə, səbəb demək olar həmişə: `sag = orta - 1` ilə `sag = orta`-nı və ya orta-nın yuvarlaqlaşmasını qarışdırmısınız. Kağızda 2-3 elementlik nümunə işlədin.

## Çalışmalar

**Çalışma 1.** Sıralanmış massivdə x-dən **kiçik ən böyük** elementi tapın (yoxdursa -1). STL ilə.

<details>
<summary>Həlli</summary>

```cpp
int i = lower_bound(a.begin(), a.end(), x) - a.begin();
// a[i] >= x olan ilk yer; ondan bir evvel — x-den kicik ən böyük
if (i == 0) cout << -1;
else cout << a[i - 1];
```

</details>

**Çalışma 2.** √n-i (tam hissəsini) yalnız vurma ilə, binar axtarışla tapın (n ≤ 10¹⁸).

<details>
<summary>Həlli</summary>

```cpp
long long n;
cin >> n;

long long sol = 0, sag = 2000000000;   // 2e9 kvadrati > 1e18
while (sol < sag) {
    long long orta = sol + (sag - sol + 1) / 2;   // YUXARI yuvarlaqla!
    if (orta <= n / orta) sol = orta;   // orta*orta <= n (dashmasiz yoxlama)
    else sag = orta - 1;
}
cout << sol;
```

İki incəlik: "ən böyük olur" axtardığımız üçün orta **yuxarı** yuvarlaqlanır (yoxsa sonsuz dövr), və `orta * orta` daşa biləcəyindən `orta <= n / orta` yazılır.

</details>

**Çalışma 3.** Array Division (CSES 1085): massivi k ardıcıl hissəyə bölün ki, hissə cəmlərinin maksimumu minimum olsun. Yoxlama funksiyasını özünüz qurun.

<details>
<summary>Həlli</summary>

```cpp
bool olur(long long limit) {
    // hər hissə cemi <= limit olmaqla neçə hissə lazimdir?
    long long cem = 0, p = 1;
    for (int x : a) {
        if (x > limit) return false;
        if (cem + x > limit) { p++; cem = x; }
        else cem += x;
    }
    return p <= k;
}
// sol = max(a), sag = sum(a) uzerinde binar axtarish
```

"Maksimumu minimallaşdır" → cavab üzərində binar axtarış + greedy yoxlama. Bu kombinasiya olimpiadalarda dönə-dönə çıxır.

</details>

## Tez-tez edilən səhvlər

1. Sıralanmamış massivdə binar axtarış.
2. `(sol + sag) / 2` daşması — `sol + (sag - sol) / 2` yazın.
3. "Ən böyük olur" axtarışında ortanı aşağı yuvarlaqlayıb sonsuz dövrə düşmək.
4. Yoxlama funksiyasında overflow.
5. `lower_bound`/`upper_bound`-u qarışdırmaq: `>=` / `>`.
