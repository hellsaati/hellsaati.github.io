# Həll saatı 7. Funksiyalar

Kod böyüdükcə onu hissələrə bölmək lazım gəlir. Funksiya ad verilmiş, təkrar çağırıla bilən kod parçasıdır. Funksiyalardan daha az səhv, daha asan debug etmək və daha sürətli kod yazmaq, yəni təkrarların qarşısını almaq üçün istifadə olunur.

Bu dərsdə öyrənəcəyik:

- Funksiyanın declare olunması, parametrlər, `return`
- Qlobal və lokal dəyişənlər
- Reference (`&`) ilə ötürmə, massivlərin ötürülməsi

## Elan və çağırış

```cpp
#include <iostream>
using namespace std;

int square(int x) {
    return x * x;
}

bool even(int x) {
    return x % 2 == 0;
}

int main() {
    cout << square(5) << '\n';
    if (even(10)) cout << "cüt";
    return 0;
}
```

Funksiyanın yazılışında funksiyanın qaytardığı tip, adı və parametrləri olaraq 3 hissəsi var, məsələn, `int square(int x)`. `return` nəticəni qaytarır və funksiyanı bitirir.

Nəticə qaytarmayan funksiya `void` tipli olur:

```cpp
void printarr(int a[], int n) {
    for (int i = 0; i < n; i++) cout << a[i] << ' ';
    cout << '\n';
}
```

:::tip[return ilə erkən funksiyanı tez bitirmək]
`return` funksiyanı istənilən yerdə bitirə bilər. Keçən dərsdəki "iç-içə dövrdən çıxma" problemi funksiya ilə gözəl həll olunur:

```cpp
bool check(int x) {
    for (int i = 0; i < n; i++)
        for (int j = i + 1; j < n; j++)
            if (a[i] + a[j] == x) return 1;  // dövr yerinə, funksiyanı birdəfəlik bitirir
    return 0;
}
```
:::

## Qlobal və lokal dəyişənlər

```cpp
int n, a[100005];
// Bu dəyişənlərə bütün funksiyalardan müraciət etmək olur və içləri avtomatik 0 olur.

void input() {
    cin >> n;
    for (int i = 0; i < n; i++) cin >> a[i];
}

int main() {
    input();
    ...
}
```

Lokal dəyişən yalnız öz funksiyasında yaşayır. Funksiyadakı `int say = 0` hər çağırışda yenidən yaranır.

:::tip
Verilənləri (n, massivlər) qlobal, köməkçi dəyişənləri isə lokal saxla. Bu, həm böyük massiv problemini həll edir [Massivlər: əsaslar](/docs/hs05.md), həm də parametrləri ötürməyi asanlaşdırır.
:::

:::danger[Adların toqquşması]
Qlobalda `n` dəyişəni varkən funksiyada `int n` elan etsəniz, lokal olan qlobalı üstələyir və funksiya daxilində lokal olan istifadə olunur. Bu, çətin tapılan səhvlərin mənbəyidir, ona görə eyni adları işlətməyin.
:::

## Qiymətlə və reference ilə ötürmə

Adi halda dəyişənin parametri kopyalanır, beləliklə funksiya daxilində dəyişiklik çöldəki qiymətə təsir etmir. `&` işarəsi isə kopyanı yox, orijinalın özünü təsir edir:

```cpp
void inc(int x) { x++; }

void inc2(int &x) { x++; }

int main() {
    int a = 5;
    inc(a);   cout << a;   // 5
    inc2(a);  cout << a;   // 6
}
```

```cpp
void minMax(int &mn, int &mx) {
    mn = a[0]; mx = a[0];
    for (int i = 1; i < n; i++) {
        if (a[i] < mn) mn = a[i];
        if (a[i] > mx) mx = a[i];
    }
}
```

Massivlər avtomatik "reference kimi" ötürülür: kopyalanmır, funksiya daxilində dəyişiklik orijinala təsir edir:

```cpp
void reset(int a[], int n) {
    for (int i = 0; i < n; i++) a[i] = 0;
}
```

`string` və gələcəkdə görəcəyimiz `vector` isə kopyalanır. Ona görə də ölçüsü böyük olduqda `&` ilə ötürün, yoxsa hər çağırış O(n) kopya ilə işləyir:

```cpp
int countA(string &s) {
    int cv = 0;
    for (int i = 0; i < (int)s.size(); i++)
        if (s[i] == 'a') cv++;
    return cv;
}
```

## Kodu funksiyalarla qurmaq

Rəqəmlərlə bağlı üç əməliyyatı funksiyalaşdıraq:

```cpp
int sum(long long n) {
    int cv = 0;
    while (n > 0) { cv += n % 10; n /= 10; }
    return cv;
}

int count(long long n) {
    if (n == 0) return 1;
    int cv = 0;
    while (n > 0) { cv++; n /= 10; }
    return cv;
}

long long reverse(long long n) {
    long long r = 0;
    while (n > 0) { r = r * 10 + n % 10; n /= 10; }
    return r;
}
```

`n % 10` ədədin təklik mərtəbəsi, `n /= 10` ədədin 10-a bölündüyünü göstərir. 


Bir sətirdə palindromu yoxlamaq:

```cpp
bool palindrome(long long n) { return n == reverse(n); }
```

## Çalışmalar

**Çalışma 1.** `pow(a, b)` funksiyasını yazın və aᵇ hesablayın (b ≥ 0, nəticə long long-a sığır).

<details>
<summary>Həlli</summary>

```cpp
long long pow(long long a, int b) {
    long long cv = 1;
    for (int i = 0; i < b; i++) cv *= a;
    return cv;
}
```

O(b) həll. Sürətli qüvvətə yüksəltmək (O(log b)) riyaziyyat dərsində gələcək.

</details>

**Çalışma 2.** İki ədədin ƏBOB-unu (GCD) tapan funksiya yazın (Evklid alqoritmi: `gcd(a, b) = gcd(b, a % b)`).

<details>
<summary>Həlli</summary>

```cpp
long long gcd(long long a, long long b) {
    while (b != 0) {
        long long r = a % b;
        a = b;
        b = r;
    }
    return a;
}
```

Gələn dərslərdə bunu dərindən öyrənəcəyik, hələlik "template" kimi yadda saxlayın. C++17-də hazır `gcd(a, b)` funksiyası da var (`#include <numeric>`).

</details>

**Çalışma 3.** `[l, r]` aralığında rəqəmləri cəmi 10-a bölünən ədədlərin sayını tapan proqramı funksiyalarla qurun.

<details>
<summary>Həlli</summary>

```cpp
#include <iostream>
using namespace std;

int sum(long long n) {
    int cv = 0;
    while (n > 0) { cv += n % 10; n /= 10; }
    return cv;
}

int main() {
    long long l, r;
    cin >> l >> r;
    int say = 0;
    for (long long i = l; i <= r; i++)
        if (sum(i) % 10 == 0) say++;
    cout << say;
    return 0;
}
```

</details>

## Tez-tez edilən səhvlər

1. `return` yazmağı unutmaq;
2. Böyük `string`/`vector`-u `&`-siz ötürüb TLE almaq;
3. Lokal dəyişəni qloballa üstələmək;
4. Funksiyanı `main`-dən *aşağıda* elan edib "not declared" xətası almaq;
5. `void` funksiyadan qiymət gözləmək.
