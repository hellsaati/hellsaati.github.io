# Həll saatı 17. Set və map

Bu günə qədər "bu element var, ya yox?" sualına massivlə dövr yazıb O(n)-də cavab verirdik. `set` və `map` bu suala **O(log n)**-də cavab verir — n = 10⁵ üçün ~17 addım. Bu iki struktur STL-in ən çox işlənən alətləridir.

## set — unikal elementlər çoxluğu

`set` elementləri **sıralanmış** və **təkrarsız** saxlayır:

```cpp
#include <set>

set<int> s;
s.insert(5);
s.insert(2);
s.insert(5);        // artiq var — hec ne olmur
s.insert(8);

cout << s.size();   // 3

//  həmişə artan sıra ilə gəzmək ilə: 2 5 8
for (int x : s) cout << x << ' ';
```

Əsas əməliyyatlar — hamısı O(log n):

| Əməliyyat | Kod | Nəticə |
|---|---|---|
| əlavə et | `s.insert(x)` | təkrar olsa dəyişmir |
| var? | `s.count(x)` | 0 və ya 1 |
| sil | `s.erase(x)` | yoxdursa heç nə |
| ölçü | `s.size()` | element sayı (O(1)) |

### Nümunə: neçə fərqli ədəd? (CSES 1621)

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    int n;
    cin >> n;

    set<int> s;
    for (int i = 0; i < n; i++) {
        int x;
        cin >> x;
        s.insert(x);
    }
    cout << s.size();
    return 0;
}
```

Təkrarları `set` özü atır — bütün məsələ 5 sətirdir. (16-cı dərsdə eyni məsələni sırala+say ilə həll etmişdik; hər ikisi O(n log n)-dir.)

## map — açar → qiymət lüğəti

`map` hər **açara** bir **qiymət** uyğunlaşdırır. Massivdən fərqi: açar istənilən tip ola bilər — böyük ədəd, string, pair.

```cpp
#include <map>

map<string, int> bal;
bal["Aysel"] = 95;
bal["Tural"] = 87;
bal["Aysel"] += 3;          // 98

cout << bal["Aysel"];       // 98
cout << bal.count("Nicat"); // 0 — yoxdur
```

:::danger[`[]` operatorunun tələsi]
`bal["Nicat"]` yazmaq (hətta yalnız oxumaq üçün!) "Nicat" açarını qiyməti `0` olmaqla **yaradır**. Yalnız yoxlamaq istəyirsinizsə `count` işlədin. Bu tələ `map`-in ölçüsünü saymaqla bağlı məsələlərdə çox insanı yandırıb.
:::

### Klassik template: tezlik sayma

"Hər ədəd neçə dəfə rast gəlinir?" — CP-nin ən çox təkrarlanan alt-məsələsi:

```cpp
int n;
cin >> n;

map<int, int> say;
for (int i = 0; i < n; i++) {
    int x;
    cin >> x;
    say[x]++;
}

// ən çox rast gelinen:
int enCox = 0, hansi = -1;
for (auto [qiymet, tezlik] : say) {
    if (tezlik > enCox) { enCox = tezlik; hansi = qiymet; }
}
cout << hansi << ' ' << enCox;
```

`auto [qiymet, tezlik]` — structured binding, `pair`-i iki ada açır. Köhnə üslubda `p.first` / `p.second` yazılır.

### Nümunə: Registration System (CF 4C)

Hər ad ilk dəfə gələndə `OK`, təkrar gələndə `ad + nömrə`:

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    int n;
    cin >> n;

    map<string, int> say;
    while (n--) {
        string ad;
        cin >> ad;
        if (say[ad] == 0) cout << "OK\n";
        else cout << ad << say[ad] << '\n';
        say[ad]++;
    }
    return 0;
}
```

Massivlə bu məsələ əzabdır; `map`-lə 8 sətir.

## lower_bound — sıralanmışlıqdan istifadə

`set`/`map` elementləri sıralı saxladığı üçün "x-dən böyük-bərabər ən kiçik element" sualına O(log n)-də cavab verir:

```cpp
set<int> s = {2, 5, 8, 12};

auto it = s.lower_bound(6);   // >= 6 olan ilk element
if (it != s.end()) cout << *it;   // 8

it = s.upper_bound(8);        // > 8 olan ilk element → 12
```

`it` — **iterator**-dur: elementə "göstərici". `*it` — qiymətin özü; `s.end()` — "tapılmadı" işarəsi. Yoxlamadan `*it` yazmayın!

:::tip[set üçün mütləq `s.lower_bound(x)`]
`lower_bound(s.begin(), s.end(), x)` yazmaq **olmaz** — set-in iteratorları üzərində bu, O(n) işləyir. Üzv funksiya `s.lower_bound(x)` isə O(log n).
:::

### Nümunə: Concert Tickets (CSES 1091)

Hər müştəri `t` qiymət təklif edir; ona qiyməti ≤ t olan **ən bahalı** bileti veririk və bilet silinir:

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    int n, m;
    cin >> n >> m;

    multiset<int> biletler;
    for (int i = 0; i < n; i++) {
        int h;
        cin >> h;
        biletler.insert(h);
    }

    while (m--) {
        int t;
        cin >> t;
        auto it = biletler.upper_bound(t);
        if (it == biletler.begin()) cout << -1 << '\n';
        else {
            --it;
            cout << *it << '\n';
            biletler.erase(it);
        }
    }
    return 0;
}
```

İki incəlik:

1. **multiset** — təkrarlara icazə verən set. Eyni qiymətli biletlər ola bilər.
2. `biletler.erase(it)` — iteratorla silmək **bir** nüsxəni silir; `biletler.erase(*it)` qiymətlə silsək, o qiymətli **bütün** biletlər silinərdi.

## unordered_set / unordered_map

Sıra lazım deyilsə, hash-əsaslı versiyalar orta halda O(1) işləyir — amma `lower_bound` yoxdur və pis halda yavaşıya bilər. Müqayisə və nə vaxt hansını seçmək 28-ci dərsin mövzusudur.

## Çalışmalar

**Çalışma 1.** n ədəd verilir. İlk dəfə təkrar görünən ədədi (ikinci dəfə rast gəldiyimiz ilk qiyməti) tapın; təkrar yoxdursa `-1`.

<details>
<summary>Həlli</summary>

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    int n;
    cin >> n;

    set<int> gorulen;
    int cavab = -1;
    for (int i = 0; i < n; i++) {
        int x;
        cin >> x;
        if (cavab == -1 and gorulen.count(x)) cavab = x;
        gorulen.insert(x);
    }
    cout << cavab;
    return 0;
}
```

</details>

**Çalışma 2.** Sözlər axını verilir. Hər söz üçün — bu söz indiyə qədər neçə dəfə görünüb (özü daxil) çap edin.

<details>

**Çalışma 3.** Sum of Two Values (CSES 1640): a[i] + a[j] = x olan i ≠ j cütü tapın. `map`-lə O(n log n) həll yazın. İpucu: hər elementə çatanda `x - a[i]` əvvəllər görünübmü?

<details>
<summary>Həlli</summary>

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    int n, x;
    cin >> n >> x;

    map<int, int> yer;   // qiymet -> indeks (1-den)
    for (int i = 1; i <= n; i++) {
        int a;
        cin >> a;
        if (yer.count(x - a)) {
            cout << yer[x - a] << ' ' << i;
            return 0;
        }
        yer[a] = i;
    }
    cout << "IMPOSSIBLE";
    return 0;
}
```

"Cütü axtarma — tamamlayıcını axtar" ideyası hashing dərsində (28) yenə qarşınıza çıxacaq.

</details>

## Tez-tez edilən səhvlər

1. `map`-də `[]`-lə yoxlayıb açarı təsadüfən yaratmaq — `count` işlədin.
2. `multiset`-də qiymətlə silib bütün nüsxələri itirmək — iteratorla silin.
3. `set`-də `lower_bound(s.begin(), s.end(), x)` yazıb O(n) almaq.
4. `*s.end()` oxumaq — undefined behavior.
5. Təkrarlar vacib olan yerdə `set` (multiset əvəzinə) işlədib məlumat itirmək.
